  `client_name`=имя организации. Переиспользует `analytics.analyze` → весь движок

  при смене пароля.

## 2026-06-18 — Источник + контроль доступа

- **РЕШЕНО (§9):** источник данных — **напрямую API Omnicomm** (не БД, не файлы).
  Один аккаунт = один фид КАП. Базис `api_client.py`+`auth.py`. Скрин Lineage от Романа
  больше не нужен — все открытые вопросы по источнику закрыты.
- **Новое требование (§8): row-level контроль доступа.** Конфиденциальность: руководитель
  видит весь КАП, ДЗО видит только своё поддерево, соседи друг друга не видят. В Power BI
  это упиралось в «лицензия на каждого ИЛИ публичная ссылка без конфиденциальности».
  Платформа делает доступ своим слоем (фильтр по поддереву `org_id`) → конфиденциальность +
  экспорт + ноль лицензий разом.
- Уточнена структура: Уранэнерго → {ТФО, ШФО}; УМЗ → {УМЗ Курчатов}.
