## 2026-06-18 — Milestone 6: чистка GPS-выбросов по физике

**Сделано (`track_clean.py`, `tests/test_track_clean.py` — 7 тестов; полный набор 169 зелёных):**
- `clean_track(track)` — межточечная физика: расстояние(haversine)/Δt → подразумеваемая
  скорость; «телепорт-и-возврат» (подход И уход невозможны, а соседи без точки согласованы)
  → точка выбрасывается, **не вся запись**. Возвращает clean/dropped_indices/plausible_max_speed.
- `reconcile_vehicle_speed(vehicle, track)` — заменяет недостоверную `max_speed_kmh` на
  правдоподобную по треку, помечает аномалию (REVIEW, не молча). Достоверную скорость не трогает.
- Это замена костыля Power BI (окно по длительности 10–90с): не теряем реальные короткие/
  длинные нарушения, режем по физике.
- Чистый модуль; **хук интеграции** — вызвать в `data_loader` где тянется трек (сейчас трек
  только для бессенсорной погрузки части ТС; fleet-wide чистка = отдельное решение).

## 2026-06-18 — Milestone 4: end-to-end оркестратор `holding.py`

**Сделано (`holding.py`, `tests/test_holding.py` — 6 тестов; полный набор 157 зелёный):**
- `build_registry(tree_nodes, …)` — реестр из дерева Omnicomm + теги подрядчиков + сохранение.
- `run(tree_nodes, vehicles, period, …)` → `HoldingRun(registry, kpi_tree, rendered,
  assigned, unassigned)` — связывает всё: реестр → ингест `org_id` → роллапы → дашборды по ДЗО

  (дерево + `data_loader.load_from_api` + `validator`), повторяет `__main__`. Импорты сети
  внутри функций → holding-слой грузится в тестах без сетевых зависимостей.
- Параметризован входом (дерево+ТС) → `run()` тестируется без аккаунта; изоляция scope и
  рендер по ДЗО покрыты.

`org_id`. Починено на `path=path`. Урок: новые параметры в середину сигнатуры → проверять
позиционные вызовы.

## 2026-06-18 — Источник + контроль доступа

- **РЕШЕНО (§9):** источник данных — **напрямую API Omnicomm** (не БД, не файлы).
  Один аккаунт = один фид КАП. Базис `api_client.py`+`auth.py`. Скрин Lineage от Романа
  больше не нужен — все открытые вопросы по источнику закрыты.
- **Новое требование (§8): row-level контроль доступа.** Конфиденциальность: руководитель
  видит весь КАП, ДЗО видит только своё поддерево, соседи друг друга не видят. В Power BI
  это упиралось в «лицензия на каждого ИЛИ публичная ссылка без конфиденциальности».
  Платформа делает доступ своим слоем (фильтр по поддереву `org_id`) → конфиденциальность +
  экспорт + ноль лицензий разом.
- Уточнена структура: Уранэнерго → {ТФО, ШФО}; УМЗ → {УМЗ Курчатов}.
