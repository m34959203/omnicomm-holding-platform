# Omnicomm Holding Platform

> Единая аналитическая платформа автопарка **холдинга КАП** (Казатомпром) поверх Omnicomm Online: собирает телеметрию по всем ДЗО из одного аккаунта, нормализует в иерархическую модель (Холдинг → ДЗО → под-ДЗО → подрядчик), переводит в деньги (₸) и раздаёт executive-дашборд на каждое ДЗО с row-level доступом — **без посемест­ных лицензий и без выгрузок вручную**.
>
> **В проде: [omnicomm.technokod.kz](https://omnicomm.technokod.kz)** — боевая копия КАП (`projectkap`): **~1962 ТС, 73 узла оргструктуры**.

Платформа выросла из [`omnicomm-fleet-report`](https://github.com/m34959203/omnicomm-fleet-report) (single-client финансовый отчёт) и переиспользует его движок (экономика топлива, типы ТС, чистка данных, моточасы спецтехники). Новое здесь — **holding-scale**: многоуровневый справочник организаций, разрезы по 20+ ДЗО, и веб-портал поверх Python-движков.

## Зачем (контекст)

Текущая отчётность холдинга собрана в Power BI как **дашборд скоростного режима** (4 страницы, ~1400 ТС, 23 ДЗО). Он хорош как safety-инструмент, но:

- **нет топливной/экономической аналитики** — главной метрики (перерасход в ₸);
- **нет моточасов и контроля датчиков** для спецтехники и буровых;
- доставка упирается в **посемест­ные лицензии Power BI** (≈8500 ₸/мес × на каждого) и **ручную выгрузку** каждому ДЗО раз в неделю;
- «широкая» модель (отдельный столбец на каждое ДЗО) не масштабируется на под-ДЗО и подрядчиков.

Платформа закрывает экономический пласт и снимает зависимость от лицензий, **переиспользуя живой поток данных** Omnicomm. Бизнес-модель — **«аналитика как услуга»**: раздача по ДЗО без посемест­ных лицензий, а платформа служит **единственным буфером к серверу Omnicomm** (раздача учёток ДЗО не множит нагрузку на сервер).

## Стек

Python-движки → FastAPI-мост → Next.js-фронт за реверс-прокси, с кэш-снапшотом в SQLite.

## Ключевые фичи

**Executive-дашборд:**
- **Scope-rail** — дерево организаций (73 узла) + поиск, **row-level доступ по ДЗО**: каждый видит только своё поддерево, начальник холдинга — весь КАП.
- **Health-полоса** — 6 KPI здоровья парка с дисциплиной цвета.
- **Лента «что требует внимания»** — топ-сигналы, **кликабельны к сути** (переход к нужному домену и подсветка ТС).
- **Обзор по доменам** + домен-табы.

**Домены:**
- **Деньги** — экономика топлива → ₸, перерасход/экономия, COI.
- **Скоростной режим СТ КАП** — превышения по геозоне × типу ТС, рекомендации «на букве закона» КоАП РК (ст.592 сверена на [adilet](https://adilet.zan.kz), ред. Закон РК 03.10.2024 №131-VIII; штраф — только для дорог общего пользования, на техдорогах — дисциплинарка).
- **Геозоны** — Яндекс-карта (~199 зон).
- **Качество данных** — терминальный Sensor Health.
- **Контроль ТО**.

**Карточка ТС** (drill-down) — трек на спутнике + график скорости + телеметрия + напряжение бортсети.

**Доставка и UX:**
- Инлайн-SVG графики (без зависимостей), **Excel-экспорт** (`/api/dashboard.xlsx`), **RU/KK**, мобильная версия.
- Выбор снимка/периода, включая **произвольные даты** по календарю.
- Тема Omnicomm Online со светлым/тёмным переключателем.

**Доверие и защита:**
- **Вентиль доверия к данным** — фильтр не-ТС (АЗС/ёмкости/ФЭС/генераторы) из денежных рейтингов (`classify.py`).
- **Защита сервера Omnicomm** — глобальный rate-limiter ≤170/мин на аккаунт (лимит Omnicomm 180/мин) + TTL-кэш карточки ТС.

## Что переиспользуется из omnicomm-fleet-report

| Возможность | Модуль |
|---|---|
| Топливо → деньги (₸), нормы, перерасход, экономия | `economics.py`, `norms.py`, `savings.py`, `fuel_price.py` |
| Типы ТС + профили расчёта (л/100км vs л/моточас) | `vehicle_types.py` |
| Чистка выбросов скорости (помечает «требует проверки», не удаляет) | `validator.py` |
| Моточасы / работа спецтехники на месте | `loading.py` |
| Omnicomm REST (auth/refresh/retry/лимиты/коды ошибок) | `api_client.py` |
| Сборка `.pptx` / `.html` | `report_builder.py`, `charts.py`, `svg_charts.py` |

## Holding-слой (новое)

- **`org.py`** — `dim_org`: иерархия организаций, построение из дерева ТС Omnicomm (`build_from_omnicomm_tree`), **row-level доступ по поддереву** (`can_view`/`visible_scope`), `filter_vehicles_for_viewer` (fail-closed).
- **`rollup.py`** — роллапы KPI по уровням иерархии (переиспользует `analytics.compute_kpi`).
- **`dashboard.py`** — дашборд/отчёт на ДЗО с изоляцией scope.
- **`holding.py`** — end-to-end оркестратор (реестр → ингест → роллапы → дашборды; забор из живого Omnicomm).
- **`store.py`** — SQLite star schema (`dim_org` + `vehicle_org`); Postgres — через `store._connect()`.
- **`geozones.py` / `speeding.py` / `recommendations.py`** — скоростной режим и рекомендации СТ КАП.
- **`track_clean.py`** — чистка GPS-трека по физике движения (телепорт-и-возврат по расстоянию/Δt).

## Запуск / разработка

```bash
# Python-движки + API-мост
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # креды Omnicomm: LOGIN/PASSWORD/SERVICE
uvicorn api.main:app --port 8810

# Веб-фронт (Next.js)
cd web
pnpm install
pnpm dev        # разработка
pnpm build      # прод-сборка

# Тесты (бинарь python3)
python3 -m pytest -q   # 274 зелёных
```

CLI holding-прогона (рендер отчётов без портала):

```bash
python -m omnicomm_report holding --demo --from 2026-05-01 --to 2026-05-31 --fuel-price 320
```
