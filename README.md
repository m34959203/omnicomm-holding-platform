# Omnicomm Holding Platform

> Единая аналитическая платформа автопарка **холдинга**: собирает телеметрию из Omnicomm Online по всем ДЗО, нормализует в иерархическую модель (Холдинг → ДЗО → под-ДЗО → подрядчик), переводит в деньги (₸) и раздаёт отчёты/дашборды на каждое ДЗО — **без посемест­ных лицензий и без логина у получателя**.

Платформа выросла из [`omnicomm-fleet-report`](https://github.com/m34959203/omnicomm-fleet-report) (single-client финансовый отчёт) и переиспользует его движок (экономика топлива, типы ТС, чистка данных, моточасы спецтехники, рассылка). Новое здесь — **holding-scale**: многоуровневый справочник организаций, мульти-аккаунтный сбор из Omnicomm и разрезы по 20+ ДЗО.

## Зачем (контекст)

Текущая отчётность холдинга собрана в Power BI как **дашборд скоростного режима** (4 страницы, ~1400 ТС, 23 ДЗО). Он хорош как safety-инструмент, но:

- **нет топливной/экономической аналитики** — главной метрики (перерасход в ₸);
- **нет моточасов и журналов акселерометра** для спецтехники и буровых;
- доставка упирается в **посемест­ные лицензии Power BI** (≈8500 ₸/мес × на каждого) и **ручную выгрузку** каждому ДЗО раз в неделю;
- «широкая» модель (отдельный столбец на каждое ДЗО) не масштабируется на под-ДЗО и подрядчиков.

Платформа закрывает экономический пласт и снимает зависимость от лицензий, **переиспользуя живой поток данных** (скрипты сбора из учётной системы холдинга → сервер в Астане, обновление 8×/сутки).

## Что переиспользуется из omnicomm-fleet-report

| Возможность | Модуль |
|---|---|
| Топливо → деньги (₸), нормы, перерасход, экономия | `economics.py`, `norms.py`, `savings.py`, `fuel_price.py` |
| Типы ТС + профили расчёта (л/100км vs л/моточас) | `vehicle_types.py` |
| Чистка выбросов скорости (помечает «требует проверки», не удаляет) | `validator.py` |
| Моточасы / работа спецтехники на месте (буровые, погрузка) | `loading.py` |
| Авто-доставка отчётов без логина у получателя | `scheduler.py`, `mailer.py` |
| Omnicomm REST (auth/refresh/retry/лимиты/коды ошибок) | `api_client.py`, `auth.py` |
| Сборка `.pptx` / `.html(+PDF)` | `report_builder.py`, `charts.py`, `svg_charts.py` |

## Что строится новым (holding-scale)

- **`dim_org`** — иерархический справочник организаций (Холдинг → ДЗО → под-ДЗО → подрядчик), с уровнем, родителем и типом (собственная / подрядная).
- **Разбор единого фида КАП по `org_id`** — маппинг ТС → организация (подрядчики и под-ДЗО в одном фиде, мульти-аккаунт не нужен — подтверждено заказчиком).
- **Нормализованное хранилище** (звезда: факты + измерения) взамен single-client JSON-персистентности.
- **Геозоны × категория ТС** — два (и более) лимита скорости на зону по типу ТС.
- **Чистка выбросов по физике движения** — валидация на приёме (расстояние/Δt между точками), а не окно по длительности.
- **Роллапы и разрезы** — агрегаты по уровням иерархии, дашборд/отчёт на любое ДЗО автоматически.
