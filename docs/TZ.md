## 0. Паспорт проекта

| Параметр | Значение |
|---|---|
| **Продукт** | Python-инструмент автоматического анализа данных автопарка Omnicomm Online → готовый клиентский отчёт уровня коммерческой презентации |
| **Заказчик** | Горкомтранс (Караганда, вывоз ТКО, ~70 мусоровозов/спецтехники) |

| **Источники данных** | Omnicomm REST API (боевой контур `kz.omnicomm.online`, демо `online.omnicomm.ru`) или Excel/CSV |
| **Форматы на выходе** | `.pptx` (основной) · `.html` (+печать в PDF) · `.xlsx` · `.pdf` (опц. LibreOffice) |
| **Валюта оценок** | Тенге (₸) |
| **Стек** | Python 3.10+ · requests · pandas · openpyxl · matplotlib · python-pptx · streamlit · playwright |
| **Персистентность** | JSON-файлы (`data/`, `output/`), без БД. Всё в `.gitignore` |

### Конвейер обработки

```
источник → data_loader → validator → analytics → charts/svg_charts → report_builder → .pptx/.html/.xlsx/.pdf
   │           │            │            │              │                  │
 API/Excel  VehicleMetrics  аномалии   FleetKPI    PNG + inline-SVG   слайды/таблицы
```

Единый контракт данных — `models.py` (`VehicleMetrics`, `FleetKPI`, `FleetReport`). Источник (API/Excel) абстрагирован в `data_loader`; остальной конвейер от источника не зависит.

# C. Новые требования этой итерации

## C1. Коэффициент использования автомобиля 🆕

**Вопрос:** как считать — ДВС / время смены? **Решение:** длины смены в телеметрии нет → ввести **нормативный фонд времени per-client** в `norms.py`/конфиг клиента. Считать две честные метрики:
- **Технический коэф.** = `engine_hours / календарный период` (всегда доступен);
- **Эксплуатационный коэф.** = `engine_hours / нормативный фонд` (если задан фонд: например 8 ч/сут или 24/7).

**Задача:** добавить поле `time_fund_hours_per_day` в конфиг клиента + расчёт в `analytics` + слайд/карточку.

## C2. Расход л/час 🆕(вывод)

`fuel_per_motorhour` уже считается (`models.py:128`). **Задача:** вывести как явную KPI-карточку и колонку (сейчас используется в основном для спецтехники/норм).

## C3. Нарушения режима труда и отдыха (РТО) ❌🆕

Непрерывные интервалы работы/движения > N часов без перерыва. **`consolidatedReport` агрегирует по суткам — оттуда не вытащить.** Требует отдельного конвейера на **по-пакетном журнале/треке**. Фаза 2. Управлять ожиданиями заказчика.

## C4. Дашборд руководителя автопарка 🆕

## C5. Особенность детекции ДВС (для Горкомтранс) 🆕

Если работа ДВС берётся **не с генератора, а по порогу напряжения** — нагрузку не видим, только время работы ДВС. Тогда:
- «Время работы ДВС без движения» (`work_no_move_hours`) — выводим честно;
- «Холостой ход» (непродуктивный) — **нельзя достоверно отделить от продуктивной погрузки** → помечать «оценка недоступна без датчика нагрузки», не натягивать.

Это уже отражено в модели: `work_no_move_hours` vs `unproductive_idle_hours`. **Открытый вопрос к заказчику:** как настроена детекция ДВС на парке.
