## Задача

Фидбэк заказчика (Дмитрий + Саша): **«информации много, очень много букв и цифр — руководители
не будут вникать. Нужны графики и диаграммы.»** Подумай, **как поднять визуальную силу графиков
в отчёте**, чтобы топ-менеджер считывал смысл за секунды. Не косметика ради косметики — каждый
график должен отвечать на управленческий вопрос («где теряем деньги», «кто вне нормы»).

Сначала ДУМАЙ и ПРЕДЛАГАЙ (с мокапами), не переписывай весь модуль. Я интегрирую после ревью.

## Что уже есть (важный контекст — ты видишь мою работу только через `git pull`)

Репозиторий: `m34959203/omnicomm-fleet-report` (public). Перед работой:
```bash
cd ~/omnicomm-fleet-report && git pull
```

- **Два выхода отчёта:**
  - `.pptx` (PowerPoint, для совещаний) — графики только **статичные PNG** (matplotlib). Интерактив тут невозможен.
  - **самодостаточный `.html`** — сейчас тоже встраивает те же PNG через base64 (`_img_data_uri`,
    `_html_figure` в `report_builder.py`). Файл открывается без сервера и печатается в PDF.
    **Здесь интерактив ВОЗМОЖЕН** (инлайн JS), но требование «самодостаточность/офлайн/печать в PDF» сохранить.
- **Где графики:** все в `src/omnicomm_report/charts.py` (matplotlib, backend Agg, DPI 150).
  Оркестратор `build_charts()`. Текущие 12 графиков (ключ → смысл):
  `mileage` (пробег по ТС), `fleet_class` (пончик подвижная/спецтехника),
  `fuel_per_100km` (расход подвижной), `fuel_per_mh` (расход спецтехники, л/мч),
  `utilization` (моточасы движение/холостой, stacked), `fuel_idle` (пончик доли простоя),
  `cost` (стоимость топлива по ТС), `money` (стэк «куда уходят деньги»),
  `overrun` (дивергентный перерасход/экономия к норме), `speeding` (превышения),
  `loading_split` (погрузка vs простой), `loading_map` (GPS-карта точек).
- **Бренд/палитра (держать):** синий `#2F5C8F` (основной), янтарь `#C8893F` (только «зоны внимания»),
  серо-синий `#7E93AB`, текст `#2B2B2B`, сетка `#D9DEE6`, фон белый. Стиль: светлый, официальный,
  без тёмных тем, без обвинительных формулировок.
- **Бизнес-инварианты (НЕ нарушать):** «сливы» клиенту не показываем; аномалии — только «требует
  проверки»; перерасход — только при заданных нормах; деньги в тенге (₸).
- **Как собрать демо-отчёт для просмотра** (есть и pptx, и html):
  ```bash
  cd ~/omnicomm-fleet-report
  PYTHONPATH=src python3 -m omnicomm_report --source excel --input samples/fleet_sample.xlsx \
      --client "Демонстрационный автопарк" --html --outdir output/demo_deck
  # → output/demo_deck/*.html — открой в браузере
  ```
- Готовые скриншоты текущих графиков: `docs/screenshots/report_*.png`.
- HTML-презентация системы (стиль, который Дмитрий одобрил): `docs/presentation/index.html`.

## Что хочу от тебя в ответе

1. **Направление** для HTML-отчёта: оставить статичные PNG но прокачать дизайн, ИЛИ перейти на
   инлайн-интерактив (какая библиотека — Chart.js / ECharts / D3 / inline SVG — с учётом
   «самодостаточный офлайн-файл + печать в PDF»). Честные плюсы/минусы, твоя рекомендация.
2. **Топ-5 графиков, которые дают максимум смысла руководителю** — и как каждый переосмыслить
   (тип, акцент, что выделять). Особенно денежные: «куда уходят деньги», перерасход к норме.
3. **Что выкинуть/слить** — если 12 графиков это перегруз, какие объединить в дашборд-сводку.
4. **Мокапы** (HTML/SVG или картинки) хотя бы для 2–3 ключевых графиков в нашей палитре.
5. Для pptx — отдельно: как улучшить статичные matplotlib-графики (типографика, подписи значений,
   убрать перегруз), оставаясь в PNG.
