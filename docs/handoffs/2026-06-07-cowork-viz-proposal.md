## TL;DR

Графики сейчас технически корректны, но «инженерные»: научная нотация на осях
(`1e6`, `1e7`), одноцветные бары без сигнала «кто проблемный», пончики ради одной
цифры, деньги без кадра «сколько можно вернуть». Руководитель не считывает смысл за
секунды — отсюда фидбэк «много букв-цифр».

**Рекомендация:** для HTML-отчёта перейти на **инлайн-SVG, генерируемый в Python
(без внешних библиотек)** + тонкий слой ванильного JS (тултипы/переключатели).
Для `.pptx` оставить matplotlib-PNG, но применить ту же визуальную систему. Не
наращивать число графиков, а ввести **сводный «экран для руководителя»** сверху и
сократить 12 разрозненных картинок до ~7 осмысленных.

## 1. Направление для HTML: статика vs интерактив

| Вариант | Плюсы | Минусы |
|---|---|---|
| **Прокачать PNG (matplotlib)** | минимум изменений; единый код для html+pptx | растр (мутно при зуме/печати); нет тултипов; «инженерный» вид сложно изжить |
| **Chart.js / ECharts (инлайн)** | быстрые красивые чарты, тултипы из коробки | canvas мутно печатается в PDF, плохо рефлоу; инлайн-бандл +200 КБ…1 МБ в КАЖДЫЙ отчёт; зависимость |
| **D3 (инлайн)** | максимум гибкости | тяжёлый, избыточен, дольше всего поддерживать |
| **Инлайн-SVG из Python (рекоменд.)** | чёткость при любом зуме; **идеальная печать в PDF**; 0 зависимостей; крошечный размер; темизация фирменной палитрой через переменные; доступность (`role/title/desc`); JS-тултипы опциональны и не ломают печать | SVG-чарты в Python писать руками дольше, чем matplotlib |

**Почему SVG, а не библиотека.** Жёсткое требование — «самодостаточный офлайн-файл +
печать в PDF». Canvas-библиотеки (Chart.js/ECharts) печатаются растром и мутнеют;
любую из них пришлось бы инлайнить в каждый отчёт (вес). SVG векторный — печатается
идеально, весит копейки, не тянет зависимостей. Интерактив (hover-тултип, переключение
«₸ / л / %», свернуть длинный список ТС) добавляется 20–30 строками ванильного JS и
**деградирует штатно**: при печати JS не нужен, весь смысл уже в самом SVG.

**Митигация трудозатрат:** маленький модуль-хелпер `svg_charts.py` (оси, бар, бар-с-
подписью, дивергентный бар, bullet, scatter) — 5–6 примитивов закрывают весь отчёт.
matplotlib остаётся только для pptx.

## 2. Топ-5 графиков, переосмысленных под руководителя

Принцип: **каждый график = один управленческий вопрос, вывод вынесен в заголовок,
янтарь только на «зоне внимания»**.

1. **«Куда уходят деньги»** (вопрос: где теряем). Сейчас — стэк с осью `1e7`.
   Стало — широкая 100%-полоса: синее «необходимое (в движении)» + янтарное
   «возвратный резерв (простой)»; **заголовок-вывод** «На простое сгорает ~7,8 млн
   ₸/мес»; подпись с годовым масштабом и суммой возврата. Ось убрать, числа — прямо
   на сегментах. → мокап №1.

2. **Холостой ход против цели** (вопрос: насколько мы вне нормы). Сейчас — пончик с
   одной цифрой. Стало — **bullet-полоса**: факт на фоне цели ≤5% и среднего по
   паркам ~33%, в деньгах ₸/мес, худшие ТС подписью. Плотность смысла кратно выше
   пончика. → мокап №2.

3. **Отклонение от нормы, ₸** (вопрос: кто вне нормы и на сколько). Оставить
   дивергентный принцип, но: **сальдо-бэйдж** сверху (перерасход vs экономия + нетто),
   затем чистый **рейтинг топ-перерасхода** (янтарь) для адресного разбора; суммы в
   «K/млн ₸», дисклеймер про нормы рядом. → мокап №3.

4. **Карта эффективности парка** (вопрос: кто аномален). **Новый** график взамен трёх
   плоских bar (`mileage` + `cost` + `fuel_per_100km`): пузырьковая диаграмма
   пробег × расход на 100 км, размер пузыря = стоимость ₸, янтарь = над нормой,
   линия нормы. Один экран показывает и загрузку, и выбросы. → мокап №4.

5. **Скоринг проблемных ТС** (вопрос: куда смотреть первым). Сейчас — только в тексте.
   Стало — горизонтальный рейтинг по баллу проблемности с чипами-причинами
   («холостой 65%», «перерасход 418К ₸»). Прямо отвечает «кого разбирать».

## 3. Что слить / выкинуть (12 → ~7)

- **Слить** `mileage` + `cost` + `fuel_per_100km` → одна **карта эффективности** (мокап №4).
- **Слить** `fleet_class` (пончик) + `utilization` (стэк) → компактная панель «состав
  и использование парка».
- **Убрать как отдельный** `fuel_idle` (пончик) — его смысл уходит в bullet холостого
  хода (мокап №2) и в «куда уходят деньги».
- **Оставить как есть** (нишевые, но нужные): `fuel_per_mh` (спецтехника),
  `loading_split` + `loading_map` (мусоровозы), `speeding` (компактно).
- **Добавить сверху** «**Экран для руководителя**»: 4–6 KPI-плиток + «куда уходят
  деньги» + bullet холостого хода — один разворот отвечает «где деньги».

Итог: **сводный дашборд + ~6 фокус-графиков** вместо 12 разрозненных.

## 5. Улучшения статичных PNG для .pptx (без смены стека)

Остаёмся в matplotlib, но:

1. **Убрать научную нотацию** (`1e6`/`1e7`) — это сейчас на `money` и `cost`. Формат
   тиков: `3,2 млн ₸` / `558 тыс ₸` (`FuncFormatter` + неразрывные пробелы).
2. **Заголовок = вывод, а не имя графика.** «На простое 2,7 млн ₸ (21%)» вместо
   «Стоимость топлива по ТС».
3. **Цвет кодирует смысл:** топ-N / над нормой — янтарь, остальное — синий, фон —
   серо-синий. Не заливать все бары одним цветом.
4. **Подписи значений на барах**, ось при этом убирать (двойное кодирование не нужно).
5. **Типографика:** крупнее шрифт, перенос длинных госномеров, меньше линий сетки,
   единый формат чисел (тыс/млн ₸, неразрывные пробелы, суффикс ₸).
6. **Пончики → bullet/полоса** там, где это «одна доля против цели».

## Ревью-заметка (Claude, 2026-06-07)

Согласен с диагнозом и с консолидацией 12→7 + «экран для руководителя» (это
ортогонально технологии рендера и однозначно правильно). По рендеру — одна важная
поправка для обсуждения с Дмитрием:

- **Cowork недооценил вариант matplotlib→SVG.** matplotlib умеет `savefig(*.svg)` —
  это вектор, печатается в PDF идеально и инлайнится в самодостаточный HTML, **без
  нового рукописного модуля** и с единым кодом для pptx(PNG)+html(SVG). Минус —
  «инженерный» вид, но его лечит именно §5-полировка (форматтеры, подписи, цвет-сигнал),
  а не смена стека. То есть §5 нужно делать в любом случае.
- **Рукописный `svg_charts.py`** даёт максимум контроля над «геройскими» графиками
  (bullet, сальдо-бэйдж, пузырьковая карта — ровно как в мокапах), но это новый код на
  поддержку. Разумен **гибрид**: §5-полишь matplotlib + SVG-вывод для HTML сразу
  (быстрый крупный выигрыш), а 2–3 «геройских» графика (деньги, bullet простоя,
  рейтинг норм) собрать рукописным инлайн-SVG там, где дизайн критичен.
- **Пузырьковая «карта эффективности»** красива, но для топ-менеджера scatter читается
  труднее ранжированного бара. Предложил бы оставить и карту, и простой топ-рейтинг
  рядом, а не заменять бары картой целиком.
