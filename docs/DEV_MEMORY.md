## 1. Что за проект

Holding-scale аналитическая платформа автопарка для холдинга **КАП** (вероятно
Казатомпром). Данные тянутся **напрямую с API Omnicomm** (один аккаунт = один фид),
нормализуются в иерархию организаций, переводятся в деньги (₸) и раздаются как
дашборды/отчёты на каждое ДЗО — **без посемест­ных лицензий Power BI**.

Выросла из single-client проекта `omnicomm-fleet-report` (его движок переиспользуется),
поверх достроен **holding-слой**.

## 2. Контекст заказчика

- Центральная учётка холдинга (**КАП**) → **23 ДЗО** → под-ДЗО → подрядчики.
  Структура реальных под-ДЗО: Уранэнерго → {ТФО, ШФО}; УМЗ → {УМЗ Курчатов}.
  **У одного ДЗО может быть несколько подорганизаций** (fan-out любой).
- Парк ~1427 ТС (1035 активны за 7 дней на момент осмотра).
- Текущая витрина — Power BI «Мониторинг скоростного режима-4» (только скорость).
  Её боли: нет топлива/экономики, нет моточасов/акселерометра, доставка через
  посемест­ные лицензии (~8500 ₸/мес × каждый) + ручная выгрузка по воскресеньям,
  «колонка-на-ДЗО» не масштабируется.
- Power BI трогать **только визуально** (правило заказчика: ничего не редактировать/сохранять).

## 3. Зафиксированные решения (НЕ переоткрывать)

| Вопрос | Решение |
|---|---|
| Источник данных | **Напрямую API Omnicomm** (не БД, не файлы). Базис `api_client.py` |
| Подрядчики/под-ДЗО | Все в **одном фиде КАП** → узлы `dim_org`, мульти-аккаунт НЕ нужен |
| Модель организаций | Иерархия `dim_org` (звезда), а не «колонка-на-ДЗО» |
| Контроль доступа | **Row-level по поддереву**: ДЗО видит свои под-ДЗО+подрядчиков, не соседей; корень=весь КАП |
| Источник иерархии | Берём из **дерева ТС Omnicomm** (узел дерева = организация) |
| Чистка выбросов скорости | По **физике движения** (расстояние/Δt), помечать не удалять — *спроектировано, не реализовано* |
| Хранилище | **SQLite** (`store.py`, star schema dim_org+vehicle_org); диспетч по расширению пути (`.db`→SQLite, `.json`→JSON). Postgres — через `store._connect()` при росте |

## 4. Карта модулей

**Holding-слой (новое в этой платформе):**
- `org.py` — `dim_org`: `Org`/`OrgTree`, построение из дерева Omnicomm
  (`build_from_omnicomm_tree`), доступ по поддереву (`can_view`/`visible_scope`),
  `assign_org_ids` (ингест), `filter_vehicles_for_viewer` (fail-closed), JSON-реестр.
- `rollup.py` — роллапы KPI по иерархии: `rollup_kpi` (org_id→FleetKPI по ТС поддерева),
  `build_org_kpi_tree` (дерево `OrgKPI`). Переиспользует `analytics.compute_kpi`.
- `dashboard.py` — дашборд на ДЗО: `build_org_report`, `render_org_report`,
  `render_for_scope` (рендер в пределах scope, изоляция ДЗО).
- `holding.py` — **end-to-end оркестратор**: `build_registry`, `run` (реестр→ингест→
  роллапы→дашборды), `fetch_fleet`/`run_from_client` (забор из живого Omnicomm). `HoldingRun`.
- `track_clean.py` — **чистка GPS-трека по физике**: `clean_track` (телепорт-и-возврат по
  расстоянию/Δt), `reconcile_vehicle_speed` (правдоподобная max_speed + пометка, не удаление).
  **Хук интеграции:** вызывать `reconcile_vehicle_speed(vm, track)` там, где тянется трек
  (`data_loader` ~стр.529), когда включат fleet-wide чистку (сейчас трек тянется только для
  бессенсорной погрузки части ТС).
- `auth.py` (расширен) — у пользователя `org_id`; `authenticate`/`user_org`/`get_user`.
- `dashboard.accessible_orgs` — организации, доступные пользователю для выбора (scope→Org list).
- `holding_app.py` (корень репо) — **Streamlit holding-портал**: вход → пользователь видит
  только свои ДЗО (scope) → дашборд выбранной организации. Запуск `streamlit run holding_app.py`.
- `store.py` — **SQLite-хранилище** реестра (star schema `dim_org`+`vehicle_org`).
  `org.save/load_org_registry` диспетчат сюда по расширению `.db`/`.sqlite`. Postgres — в `_connect()`.
- `models.VehicleMetrics.org_id` — привязка ТС к узлу.

**Унаследованный движок (single-client, переиспользуется как есть):**
- `api_client.py`/`auth`(Omnicomm JWT в api_client)/`data_loader.py` — забор из Omnicomm/Excel.
- `validator.py` — чистка/аномалии (порог `MAX_PLAUSIBLE_SPEED_KMH`).
- `analytics.py` — `compute_kpi`, `analyze` → `FleetReport`.
- `economics`/`norms`/`savings`/`fuel_price` — топливо→₸, перерасход.
- `vehicle_types.py` — типы ТС + профили. `loading.py` — моточасы спецтехники.
- `charts.py`/`report_builder.py` — графики + `.pptx`/`.html`. `scheduler`/`mailer` — доставка.

## 5а. Продуктовые требования КАП (решения 22.06 — `holding-product-requirements.md`)

- **Доступ:** row-level по ДЗО сохраняется (начальник=весь КАП, ДЗО=своё поддерево), **тонкие
  роли НЕ нужны** — всем одинаковый дашборд (просмотр+экспорт) в своём scope; у каждого ДЗО свой
  логин/пароль. В `auth.py` уже почти так (admin / manager+org_id). Не усложнять RBAC.
- **MVP-приоритеты директора:** (1) Excel-экспорт [главное] (2) RU/KK (3) мобильная (4) логин на ДЗО
  [есть] (5) настраиваемая частота обновления.
- **Ограничение:** видеотерминалы НЕ Omnicomm, в Omnicomm не передают → видео/ADAS/DSM = отдельный
  контур, не Omnicomm API. Профильные системы Казатомпрома недоступны.
- **База знаний `docs/knowledge-base/`** (6 файлов): каталог датчиков, ADAS/DSM, право РК (КоАП/ПДД,
  МРП 2026=4325₸), таксономия ТС + `geozone_limit`, движок рекомендаций «на букве закона».

## 5в. Чек-поинт 2026-06-23 (прод-стек + большой пакет фич)

Прод: **Python-движки → FastAPI (`api/`) → Next.js (`web/`)** за реверс-прокси (`deploy/holding_proxy.py`),
кэш-снапшот SQLite (`api/cache.py`). Полный реестр модулей/эндпоинтов/компонентов — в **[CLAUDE.md](../CLAUDE.md)**.

- **Дашборд переработан** (панель дизайн-экспертов): из стопки 6 равновесных секций → executive-вид.
  `ScopeRail` (дерево 73 узлов + поиск, row-level скоупинг всего по `vehicle_org`), `HealthStrip`
  (6 health-KPI, дисциплина цвета), `AttentionFeed` (топ-сигналы, **кликабельны к сути** → таб + подсветка
  ТС), `Overview` (4 домен-карточки), `DomainTabs` (Деньги/Скорость/Геозоны/Качество/ТО, карта lazy).
- **Инлайн-SVG графики** (`web/.../charts.tsx`, без зависимостей): `ColumnChart` (гистограмма тяжести
  6-20/20-40/40+), `RankBars` (топ-ДЗО), `StackedBar` (статусы), `LineChart` (скорость во времени).
- **Вентиль доверия** (`classify.py` `is_transport`): не-ТС (АЗС/ёмкости/ФЭС/генераторы/IMEI) убраны из
  денежных рейтингов и рекомендаций (экономика считается по транспорту). Убрало «солнечная станция жжёт ₸».
- **Выбор снимка**: dropdown снимков + шаблоны периода (Сутки/2 суток/Неделя) + **произвольный период по
  календарю** (с/по даты). `SyncBar`; `/api/sync` принимает `start_ts/end_ts`.
- **Карточка ТС** (`api/vehicle.py` + `VehicleCard.tsx`): drill-down «Трек на карте» → модал с Яндекс-гибридом
  (спутник), полилиния трека (синяя), маркер ТС, график скорости (зелёный), телеметрия. `/api/vehicle/{id}`
  (трек, быстро ~2с) + `/api/vehicle/{id}/telemetry` (сводный ~16с, лениво). TTL-кэш (трек 5 мин / телеметрия 15 мин).
- **Напряжение бортсети** — `api_client.get_vehicle_state` → `GET /ls/api/v1/vehicles/{id}/state`: `voltage`
  (+ адрес, зажигание, тек. скорость/топливо). ВАЖНО: voltage **НЕ в** `consolidatedReport`. **Снимает блокер
  Sensor Health ур.1.5** («сбой ДУТ vs обесточенный терминал»). Док — `docs/api.md §16.5`.
- **Структурные рекомендации** — канон [kb-05](knowledge-base/05-recommendation-engine.md): Нарушение → Частота →
  Квалификация → Штраф/ставка → Ущерб → Действие (размеченные поля вместо абзаца).
- **Сверка ст.592 КоАП на adilet** — ред. **Закон РК 03.10.2024 № 131-VIII**; таблица 5/10/20/40 МРП верна.
  `KOAP_VERIFIED=True`, **R-INV-8 снят**: суммы штрафов показываются для дорог общего пользования (техдороги —
  дисциплинарка). `kb-03` помечен «сверено».
- **Тема Omnicomm Online** — палитра (navy `#13294b`, синий `#1f6fd6`, зелёный `#2e9e5b`) + Roboto; светлая
  (Omnicomm) по умолчанию, тёмная (navy) через `ThemeToggle`, токены `[data-theme]`, анти-флэш.
- **Защита сервера Omnicomm** — `rate_limit.py`: глобальный token-bucket **≤170/мин на аккаунт** (лимит Omnicomm
  180/мин), общий на все потоки синка, в `OmnicommClient._throttle`. **Принцип: платформа = единственный буфер
  к Omnicomm — раздача ДЗО-учёток НЕ множит нагрузку** (ДЗО читают из кэша, фильтр по scope; синк один за всех).
- Карта геозон — Яндекс.Карты (hybrid) основной рендерер (`lib/ymaps.ts`), MapLibre/Google фолбэки.
- Sensor Health (R7.1/2), Контроль ТО (R6.1-3), Excel-экспорт, RU/KK i18n, мобильный — в проде (`api/health.py`,
  `api/excel.py`).
