# Экспертный разбор решения — 2026-06-22

> Анализ концепции/ТЗ/технического решения тремя независимыми экспертами (системный архитектор, продукт/бизнес, риск+право РК+качество данных). Цель — найти дыры до реализации. Здесь — консолидированные находки с приоритетом и владельцем. Правки по части пунктов уже внесены в [SOLUTION](SOLUTION.md)/[TZ](TZ-holding.md)/[kb-04](knowledge-base/04-vehicle-taxonomy-and-speed-policy.md)/[kb-05](knowledge-base/05-recommendation-engine.md) (отмечено ✏️).

## Красные линии (инварианты продукта) — ОБЯЗАТЕЛЬНЫ
Внесены в [kb-05 §0](knowledge-base/05-recommendation-engine.md). Нарушение = блокер релиза.
1. **R-INV-1 (право дороги).** Статья КоАП + сумма в ₸ — **только** для геозон «дорога общего пользования» (Зоны ПДД 5/6, магистрали `M-/A-/P-`, н.п.). Технологические дороги/ГТП/полигоны/промплощадки (Зоны 1–4) → **только дисциплинарное отклонение по СТ КАП**, без статьи и без ₸. ✏️
2. **R-INV-2 (нет фиктивных сумм).** Запрет персональной «суммарной задолженности по штрафам» к водителю. Деньги — только агрегированный обезличенный риск-индикатор с оговоркой «оценочно, не начисленный штраф». ✏️
3. **R-INV-3 (достоверность скорости).** Превышение — только по **устойчивому сегменту** трека (≥N правдоподобных точек + фильтр ускорения Δv/Δt), не по одной точке; для денег планка достоверности выше, чем для дисциплинарного флага. ✏️ ([kb-04 §3.3](knowledge-base/04-vehicle-taxonomy-and-speed-policy.md))
4. **R-INV-4 (вентиль качества).** ТС с непроверенным/«грубо»-валидным ДУТ — не в денежные рейтинги без confidence-флага.
5. **R-INV-5 (verified geozone seed).** Квалификация только по отвалидированному seed геозон (без дублей-конфликтов/опечаток/чужих имён); геозоны без чистого имя-матча к Omnicomm — не квалифицируются деньгами.
6. **R-INV-6 (нет авто-кадровых решений).** Платформа информирует, не предписывает депремирование/приказ; формулировки — «к рассмотрению руководителем».
7. **R-INV-7 (ПДн).** Досье водителя — только при правовом основании, минимизации полей, без авто-негативных решений по возрасту.
8. **R-INV-8 (сверенный закон).** Суммы — только по статьям со статусом «сверено с adilet» + `koap_revision_date` в конфиге.

## P1 — до прод-релиза
- **Идемпотентность фактов** при 8×/сутки + пересчётах Omnicomm: natural key `(terminal_id, date[, …])` + UPSERT + `source_loaded_at`. (Арх)
- **Наработка ТО = SUM(факт) от T0**, не инкремент-счётчик (иначе дрейф/двойной счёт). **T0-offset** (стартовая наработка) при онбординге, иначе ложные ТО-алерты первый месяц. (Арх+Риск+Прод)
- **vehicle_org без истории/категории** расходится с north-star (`valid_from/valid_to`, `category`): при переводе ТС/смене категории KPI и лимит едут задним числом. Версионировать или явно вынести из MVP. (Арх)
- **Push-доставка** (еженедельный дайджест ДЗО: топ-3 проблемных ТС + потери ₸ + действие); дашборд — для drill-down. (Прод, adoption)
- **Разделить «правовое/точное» (штраф) и «оценочное/вероятностное» (ущерб ДТП)**; не суммировать. (Прод+Риск)
- **Фрейминг для ДЗО — «ваш инструмент экономии», не «контроль сверху»**; рейтинг/лига ДЗО как мотиватор. (Прод)
- **Сверить КоАП** (ст. 590–600/571/592, скидка 50%) с adilet; `koap_revision_date`; скрывать несверённые составы (R-INV-8). (Риск)
- **ADR-движок выключен** до списка Данияра + сверки редакции Правил от 25.04.2026. (Риск)
- **Shadow-период детекции**: считать без публикации денежных карточек до валидации precision заказчиком. (Риск)
- **Трансграничность:** `online.omnicomm.ru` — РФ-домен; подтвердить геолокацию копии и каналов (playwright/SMTP) — реальная, не декларативная не-трансграничность. (Риск)
- **Health-check учётки** (права + доступность эндпоинтов; каталог уже отдаёт `permissions`, «Журнал» 404) + алерт при изменении прав; SLA на копию КАП. (Риск+Арх)
- **Роль «ответственный за ТО»** — это запись (подтверждение ТО), конфликтует с read-only-моделью. Развести. (Прод)
- **Методология расчётов** как клиентский артефакт (как считаем перерасход/моточас/потери). (Прод)
- **Sensor registry.status** — без сверки с фидом ведётся вручную; в TZ R7.3 развести «реестр (ручной)» и «статус (недоступен без Журнала)». ✏️
- **Цифра парка 1427 vs 2000** — свести в одном месте; бюджеты запросов пересчитать под ~2000. ✏️

## P2 — позже / на масштабе
- Досье водителя, реестр серийников, спец-лимиты по грузам — отложить (зависят от данных, которых нет).
- Аудит-лог доступа; вынос ПДн досье в отдельный доступ-логируемый раздел/под-роль.
- Роллап не масштабировать наивно при добавлении факт-джойнов; порог Postgres — **до** запуска факт-таблиц (violations/fuel), не «при росте».
- `category` ТС присвоить до использования фолбэк-матрицы в денежной квалификации.
- МРП/статьи версионировать по году (нарушение прошлого периода — по МРП того года).
- Upsell-обоснование (DSM/ADAS) — только на верифицированной статистике (конфликт интересов на ложных срабатываниях).
- Инвариант-чек привязки на ингесте: сколько ТС из дерева не получили `org_id` → алерт, а не тихий fail-closed дроп.

## Открытые вопросы к людям (для встречи)
- **Дима:** плательщик за аналитику (КАП централизованно / ДЗО) + модель цены на 2000 ТС; revenue-share по upsell DSM/ADAS; формат и периодичность холдингового отчёта наверх.
- **Антон:** таксономия ТС (блокер `geozone_limit` по типу); формат боевого фида; доступ к «Журналу»/расширенные права Omnicomm; события DMS/ADAS как доп-источник.
- **Данияр:** список опасных грузов (ADR).
- **Внутреннее/юр:** сверка статей КоАП на 2026; правовое основание ПДн досье; геолокация хранилища копии.

## Что в решении уже хорошо (не ломать)
Row-level fail-closed изоляция (`org.py`); единый контракт `models.py`; честная фиксация ограничений API («Журнал» 404, нет серийников/напряжения); каталог аномалий геозон (09 §5); Sensor Health как вентиль перед аналитикой; инвариант «без обвинений / требует проверки»; позиционирование vs IT-Transport; «иерархия вместо колонки на ДЗО». Главный вывод аудита: **движок должен ИСПОЛНЯТЬ правовые и качественные оговорки, которые уже сформулированы в базе знаний, но обходятся в алгоритме.**
