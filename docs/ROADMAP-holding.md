## Фаза 2 — Ядро ценности: скоростной режим «на букве закона»

- ⛔ **Seed-датасет геозон** из СТ КАП (~400, имя→ДЗО→лимит→тип) — verified-артефакт с чисткой аномалий (Заречное с чужими именами, дубли 40/80, опечатки, СТОП=0). [P0-B]
- 🟡 **`geozone_limit(геозона, тип_ТС)`** — ✅ движок `geozones.py` (матрица 6×3, классификация риск-зоны по имени, гейт «КоАП vs дисциплинарка», дефолтный классификатор категории ТС, именованный лимит = источник истины, парсинг «NN км/ч»). 10 тестов; проверен на 200 боевых геозонах. ⛔ осталось: полный `geozone_limit` с привязкой **точка→геозона по геометрии** + полный seed-справочник (~400 строк) + матчинг к Omnicomm по `(owner_org+норм.имя)`.
- 🟡 **Детекция превышений** — в проде детекция по **визитам геозон** (430 ТС КАП); графики: гистограмма тяжести, топ-ДЗО. ⛔ осталось: детекция по **сырому треку** (`speeding.py` написан/протестирован, но привязки точки→геозона по геометрии в проде ещё нет).
- ✅ **Движок рекомендаций** (`recommendations.py`) — подключён, рекомендации СТ КАП в проде. Структурированы по канону kb-05 (Нарушение→Частота→Квалификация→Штраф→Ущерб→Действие).
- ✅ **Справочник статей/МРП в `config.py`** (ст.592, МРП 2026). **Сверка КоАП ст.592 на adilet ВЫПОЛНЕНА** (ред. Закон РК 03.10.2024 №131-VIII): `KOAP_VERIFIED=True`, R-INV-8 снят — суммы штрафов показываются для дорог общего пользования (на техдорогах — дисциплинарка).
- ⛔ **Shadow-период** — считать нарушения без публикации денежных карточек до валидации precision заказчиком.

## Фаза 4 — Прод, масштаб, данные

- 🟡 **Идемпотентность фактов** — ✅ `violation` (natural key + UPSERT + source_loaded_at); ⛔ осталось `fact_fuel`/`fact_events` той же формой.
- ⛔ **Postgres до запуска факт-таблиц** (не «при росте»); порог перехода — перед violations/fuel.
- ⛔ **Версионирование `vehicle_org`** (`valid_from/valid_to`, `category`) — иначе KPI/лимит едут задним числом при переводе ТС.
- 🟡 **Health-check учётки** (`account_health.py`) — реализован и проверен вживую, но **не вызывается** из прогона (утилита на полке).
- 🟡 **Экономика — разделить «измеримо» vs «оценка»** + гард на аннуализацию `<7 сут` (COI завышается из 2-дневного окна — не аннуализировать короткие срезы).
- 🔴 **События Omnicomm** (`events/{id}`) на `projectkap` **ПУСТЫ** — нужен provisioning событий или другая учётка с доступом; блокирует DMS/ADAS как доп-источник.
- 🟡 **Прод-онбординг как процесс:** крипто-пароли (задать `APP_CRYPTO_KEY` в прод-`.env`), приёмочный тест изоляции ДЗО, переход на Postgres при росте, хранение сырья (~50–60 ГБ/год треки на 4 ТБ — с запасом).
- ⛔ **ПДн/трансграничность** — подтвердить геолокацию копии (`online.omnicomm.ru` — РФ-домен) и каналов; правовое основание досье водителя. [R-INV-7] 🔴
- ⛔ **Досье водителя** (год рождения/стаж/ДОПОГ) — после правового основания; требует идентификации (iButton/RFID, в фиде есть `DRIVER_CODE`).
- ⛔ **Деплой** — cron+flock-вотчдог, прокси перед Streamlit, мониторинг (как в fleet-report); инфра в РК.
- ⛔ **Расширенный «Журнал»/ретрансляция** (EGTS/Wialon) — для сенсор-потока в реальном времени (опц., путь B [kb-10]).
