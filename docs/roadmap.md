## Модель запуска и контроля (важно)

Сейчас система **не демон** — запускается по требованию: (1) платформа Streamlit —
человек выбрал клиента и нажал «Сформировать отчёт»; (2) CLI `python -m omnicomm_report
--preset ... --email ... --alert-email ...` — один прогон. Автономную периодическую
рассылку даёт **внешний планировщик (cron на сервере)**, запускающий ту же CLI-команду.
Контроль: API только на чтение (ничего не меняет в парке); снапшоты истории
идемпотентны на период; SMTP-сбой/нет данных не валят прогон; лимиты Omnicomm
соблюдаются клиентом. Встроенный планировщик-демон — в бэклоге выше.
