# Продукт

## Что это

Инструмент автоматического анализа данных автопарка. На вход — данные из Omnicomm Online (REST API) или Excel-выгрузка; на выходе — готовый клиентский отчёт в PowerPoint уровня коммерческой презентации.

## Для кого

- Менеджеры/аналитики, которые сейчас вручную собирают отчёты по автопарку.
- Руководство и конечные клиенты, которым нужен читаемый аналитический отчёт, а не техническая выгрузка.

## Что считает (ТЗ §5)

- общий пробег;
- общий расход топлива;
- средневзвешенный расход на 100 км;
- общее количество моточасов;
- долю топлива, израсходованного без движения;
- долю пробега с превышением скорости;
- ТС с максимальным расходом и с максимальным числом отклонений;
- аномальные значения скорости.

## Принципы отчёта (ТЗ §6, §7, §9)

- Управленческие выводы, а не «голые» цифры.
- Официальный стиль, без разговорных и обвинительных формулировок.
- Аномалии помечаются как **«требуют проверки»**, без подтверждённых обвинений.
- Столбец «возможные сливы топлива» в итоговый отчёт **не выводится**.
- Перерасход топлива не утверждается без согласованных норм.

## Структура презентации (ТЗ §8)

1. Титульный слайд — название, период, кол-во ТС, ключевые KPI.
2. Ключевые выводы для руководства.
3. Профиль автопарка (таблица по ТС).
4. Пробег и распределение нагрузки.
5. Топливная эффективность.
6. Расход без движения.
7. Операционные отклонения.
8. Рекомендации и план действий.

## Пользовательский сценарий (ТЗ §13)

1. Выбрать источник (Omnicomm API или Excel).
2. Указать период, при необходимости — список ТС и название клиента.
3. Запустить формирование отчёта.
4. Скачать готовый `.pptx` (или `.pdf`).
