# Методика денежных цифр: что измерение, а что оценка

Документ для честной коммуникации с заказчиком. Принцип: **измеренное и
оценочное разделены и помечены**. Любая строка с «≈» в отчёте — оценка по
отраслевому коэффициенту, а не факт.

## Три уровня достоверности

| Уровень | Что это | Где в отчёте |
|---|---|---|
| **Измерение** | Прямо из телеметрии Omnicomm (литры, моточасы, пробег × цена ГСМ) | KPI, профиль парка, «холостой ход (факт)», «перерасход к нормам» |
| **Verified savings** | Факт против замороженного baseline по методике IPMVP-lite | Слайд «Счётчик экономии против baseline» |
| **Оценка** | Отраслевой коэффициент × телеметрия (помечено «≈») | Слайд «Экономический эффект»: износ/ТО, эко-вождение, COI ₸/мес |

## Зашитые коэффициенты (`config.py`)

| Константа | Значение | Назначение | Источник | Тип |
|---|---|---|---|---|
| `IDLE_REDUCIBLE_SHARE` | 0.30 | какая доля простоя практически сокращаема | бенчмарк 40–60%/30 дн, берём осторожные 30% | оценка |
| `IDLE_WEAR_KM_PER_HOUR` | 30 | 1 ч холостого ≈ N км износа двигателя | Cummins severe-duty | оценка |
| `MAINT_COST_PER_KM_KZT` | 40 | стоимость ТО на км | ATRI 2025 (~$0.123/км), для КЗ ниже; ENV-override | оценка |
| `ECO_DRIVING_SAVE_SHARE` | 0.05 | эффект эко-вождения на топливо в движении | исследования 5–10%, берём нижнюю | оценка |
| `NORM_COEFFICIENTS[winter]` | 1.10 | зимняя надбавка к норме/ожиданию | Минтранс РК ~6–12% | оценка |
| `IDLE_TARGET_SHARE` | 0.05 | цель по доле холостого хода | Geotab benchmark | справочная |

Все оценочные коэффициенты вынесены в `config.py` и переопределяемы
per-client при появлении фактических данных (например, реальная ставка ТО).

## Слайд «Экономический эффект» — построчно

- **Топливо: холостой ход** — *измерение*, если есть датчики надстройки
  (непродуктивный простой по факту); иначе *оценка* (`IDLE_REDUCIBLE_SHARE`
  от стоимости топлива без движения), помечается «≈».
- **Топливо: перерасход к нормам** — *измерение* (факт сверх утверждённых
  норм; только при заданных нормах — бизнес-инвариант).
- **Износ и ТО** — *оценка* «≈» (`IDLE_WEAR_KM_PER_HOUR × MAINT_COST_PER_KM_KZT`).
- **Вождение и режим** — *оценка* «≈», только потенциал (`ECO_DRIVING_SAVE_SHARE`).
- **COI ₸/мес, ₸/год** — *экстраполяция* периода (×30/12), не измеренный
  результат. Считается только на периоде ≥ 7 дней (короче — шум ×30).

Формулировка на слайде: «оценка потерь и потенциала», не «вы потеряли».

## Слайд «Счётчик экономии против baseline» — IPMVP-lite

Это **измерение факта против эталона**, не прогноз:

```
ожидание = ставки baseline × ФАКТИЧЕСКАЯ активность периода (± сезон)
экономия = ожидание − факт  (литры → ₸ по цене периода)
```

Ограничения, помеченные в отчёте:
- работает только при **явно замороженном** baseline (контрактная точка);
- **прочность baseline = число периодов-источников**. Baseline из 1 периода
  → пометка «оценка предварительная»; точность растёт с историей;
- нормализация по фактической активности: «ездили меньше» экономией не
  считается;
- **защита от двойного счёта**: пересекающиеся периоды (напр. ручной блок
  01–12 поверх дневных) исключаются из накопленного итога —
  `savings._dedupe_overlaps` (регресс-тест в `tests/test_savings.py`).

## Что НЕ заявляем

- Перерасход без утверждённых норм — не выводится.
- «Подтверждённая экономия» как маркетинговый термин не используется, пока
  нет ≥ 2 периодов baseline и нескольких отработанных периодов программы.
- Оценочные суммы не складываются с измеренными в один «итог сэкономлено».
