02Проблема

Power BI отвечает на один вопрос из десяти

  • Только скорость. Витрина «Мониторинг скоростного режима» — 4 страницы; нет топливной экономики (₸), КПД спецтехники, контроля ТО и качества данных.
  • Дорогая ручная доставка. Посемест­ные лицензии ≈ 8 500 ₸/мес × получатель + ручная выгрузка каждому ДЗО раз в неделю.
  • Модель не масштабируется. «Колонка на каждое ДЗО» ломается при добавлении ДЗО / под-ДЗО / подрядчика.
  • Рекомендации — «ИИ-вода». Общий план без статьи закона, суммы штрафа и конкретного действия.
Omnicomm Holding Platform02
03Решение

Единый «мозг автопарка» поверх Omnicomm

Платформа собирает телеметрию всех ДЗО через API, нормализует в иерархию организаций (Холдинг → ДЗО → под-ДЗО → подрядчик), переводит километры и литры в деньги и раздаёт каждому ДЗО свой дашборд — без посемест­ных лицензий, с доступом строго к своей технике.

Платформа не переизобретает расчёты Omnicomm и не лезет в видео-железо — её ценность это слой, которого нет ни у Omnicomm, ни у Power BI: кросс-ДЗО агрегация + экономика + рекомендации на букве закона.

08Качество данных

Доверие к цифрам — отдельный слой

  • Светофор терминалов. Online / устарели / офлайн по давности данных — стек-бар с долями.
  • Пропавшие блоки. ТС без топлива/CAN/оборотов помечаются → не врут в KPI.
  • Честная граница. Сенсор-уровень (напряжение/Журнал) недоступен по REST — так и написано, без overclaim.
Omnicomm Holding Platform08
09Доставка

Каждому ДЗО — своё, без лицензий

  • Row-level доступ. Руководство видит весь КАП, ДЗО — только своё поддерево; соседи не видят друг друга.
  • Excel одной кнопкой. Текущий срез — 6 листов из снапшота.
  • Два языка RU / KK и мобильная версия.
  • Выбор снимка + шаблоны периода. Сутки / 2 суток / неделя — один клик.
Omnicomm Holding Platform09
10Архитектура быстро и масштабируемо

Python-движки → FastAPI → Next.js

  • Python-движки. Нормализация, экономика, скоростной режим СТ КАП, роллапы по иерархии — отлаженное ядро.
  • FastAPI-мост + кэш-снапшот. Синхронизация с Omnicomm идёт фоном; чтения дашборда отдаются из кэша мгновенно и не дёргают Omnicomm.
  • Next.js-интерфейс. Дашборд ДЗО, инлайн-SVG графики, карта геозон, рекомендации; RU/KK; статический экспорт за реверс-прокси.
  • Устойчивость. Синк изолирован: его сбой не роняет сервер; данные переживают перезапуск. Вся инфраструктура — в РК.
