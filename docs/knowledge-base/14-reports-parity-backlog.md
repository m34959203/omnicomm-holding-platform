## Очередь реализации

- **P1 (популярные + дыры, делаем первыми):**
  1. **Объём топлива** — график уровня топлива по ТС (данные есть, нужен ряд+рендер).
  2. **Заправки и сливы** — форма событий заправок/сливов (мульти-ТС, xls).
  3. **Местоположение** — fleet-wide карта живых позиций (поверх `get_vehicle_state`/архива).
  4. **Нарушения** — расширить детекцию (стоянки/обороты/ДУТ) сверх скоростного режима.
  5. **Посещение геозон** — табличная форма визитов (данные уже тянем).
- **P2 (детализация по ТС/сменам):** Движение, Стоянки, Работа группы, Посменный, События
  (События/Журнал — упираются в права REST; запросить учётку с провиженингом событий).
- **P3 (ниша/блок доступа/вне контура):** Выдача топлива, АТЗ-выдачи, Журнал (сырьё), Мультимедиа.

## Сквозные требования (из аудита — добавить ко всем нашим отчётам)
- **Экспорт XLS** на каждую табличную форму (у нас есть `/api/dashboard.xlsx` — распространить).
- **Печать/PDF** табличных форм.
- **Группы колонок** (вкладки внутри отчёта) как в «Сводном»/«Работе группы»/«Стоянках».
- **Пресеты периода** `сегодня/вчера/7д/месяц` + произвольный — у нас есть в `SyncBar`, унифицировать.
- **«Зафиксировать» (снапшот)** — у нас уже снапшот-кэш, можно дать кнопку фиксации в UI.
- **Рабочие столы** (карта+отчёт рядом) — паттерн Omnicomm, рассмотреть как пресет-композиции.

## Замечания по доступности (НЕ переоткрывать)
- `events/{id}` и «Журнал» по REST на `projectkap` закрыты (404/пусто) — полные «События»/«Журнал»
  требуют учётки с правами; на текущей учётке держим эвристики (геозон-визиты, агрегаты).
- Видео/фото (Мультимедиа) — отдельный контур, в Omnicomm у КАП не передаётся.
- Часть тяжёлых форм («Посещение геозон», «Посменный») в самом Omnicomm строятся долго на
  большой выборке — у нас это снимается снапшот-кэшем (считаем синком, отдаём мгновенно).
