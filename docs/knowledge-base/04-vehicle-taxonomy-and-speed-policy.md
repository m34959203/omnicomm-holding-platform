# Таксономия типов ТС и политика скоростных лимитов

> Синтез вводных директора по развитию Omnicomm (аудио 22.06.2026) + право РК ([03](03-legal-framework-kz.md)) + реальная структура парка копии КАП.
> Это спецификация для `geozone_limit(geozone, category, max_speed)` и категоризации ТС в движке.

---

## 1. Зачем своя таксономия

Omnicomm/Power BI знают **один** лимит на геозону (или вообще общий). Реальность холдинга: **лимит зависит от типа ТС × типа дороги/зоны**. Буровой КрАЗ по пересечённой местности и легковой на трассе — разный допустимый режим. Это прямой value-add платформы (north-star `geozone_limit`), которого у конкурента нет.

Прямая цитата директора: *«…КрАЗы, Уралы, которые в принципе просто выходят из строя из-за превышения скоростных режимов, особенно по пересечённой местности… как у ИНКАЯ, у них должна быть максимально разрешённая скорость 50 км/ч»*.

## 2. Таксономия типов ТС (из вводных директора)

Согласовать финальный список с **Данияром** (он же даёт список опасных грузов) и **Антоном**.

| Класс | Примеры | Ключевой риск | Профиль лимита |
|---|---|---|---|
| **Опасные грузы (ADR)** | топливозаправщики, химия, ВВ (классы 1/6/7) | катастрофа + регуляторный (лицензия, ДОПОГ) | строгий; маршрут согласован с УДП; см. [03 §3](03-legal-framework-kz.md) |
| **Шаланды / полуприцепы** | седельные тягачи с п/прицепом, контейнеровозы | занос, опрокидывание, расцепка | «грузовые > 3,5 т с прицепом» → вне НП 70–90 км/ч |
| **Пассажирские (легковые)** | вахтовые легковые, перевозящие людей | риск для людей | как легковые, но строже по поведению водителя |
| **Автобусы (малотоннажные и пр.)** | вахтовые автобусы, микроавтобусы | риск для людей (много) | автобусы: 90 вне НП, 60 в НП; усиленный DSM |
| **Тяжёлая/спецтехника (КрАЗ, Урал)** | бурение, самосвалы, вездеходы | **износ/поломка от превышения по пересечёнке** | технологические дороги — **50 км/ч (внутр. стандарт)** |
| **Лёгкий коммерческий** | Prado, пикапы, легковые ИТР | стандартный | по ПДД + геозона |

Эта таксономия ложится поверх существующего `vehicle_types.py` (типы + профили расчёта). Добавить поле `category` с историей (north-star `dim_vehicle.category`) и связать с лимитами.

## 3. Политика скоростных лимитов

### 3.1. Модель `geozone_limit(geozone, category) → max_speed`

> ✅ **Стандарт уже существует и получен** — СТ АО НАК «Казатомпром» + экспорт ~400 именованных геозон с лимитами. Полный разбор и данные — **[09-kap-geozone-speed-standard.md](09-kap-geozone-speed-standard.md)**. Ниже — модель, которую он задаёт.

Лимит = **функция (геозона, категория ТС)**, с приоритетом «меньшее из»:

```
лимит(ТС, точка) = min(
    лимит_ПДД(категория, тип_дороги),         # закон РК, раздел 10 ПДД [03]
    лимит_геозоны(геозона[, тип_ТС]),         # ИСТОЧНИК ИСТИНЫ: именованные геозоны КАП [09]
    зонная_матрица(категория, зона_риска)     # фолбэк вне именованной геозоны [09 §1.А]
)
```

- **Источник истины — именованные геозоны КАП** (~400 шт., имя→лимит, [09 §3](09-kap-geozone-speed-standard.md)). Имя совпадает с Omnicomm (Prado: «Дорога до Рудник Инкай 80 км/ч»). Координат в справочнике нет — геометрию тянуть из Omnicomm по имени.
- **Зонная матрица (фолбэк)** — 6 зон риска × 3 категории ТС, единственная даёт разбивку по типу ТС ([09 §1.А](09-kap-geozone-speed-standard.md)). Это и есть «внутренний стандарт» вместо моей прежней догадки «50 км/ч»: реальные лимиты техдорог/полигонов КАП = 5–50 км/ч по зоне.
- **Жёсткое правило приоритета (п. 7.3/7.7 СТ КАП):** `min(внутренний, ПДД)`; внутренний лимит превышать нельзя, даже если ПДД допускают больше.
- **Пороги нарушения СТ КАП:** до 3 км/ч — незначительное · 3–5 — существенное · **≥6 — грубое** (строже КоАП, где штраф от 10 км/ч — это дисциплинарка холдинга, не штраф).

### 3.2. Базовые лимиты ПДД РК (раздел 10) — встроенный справочник

| Категория | Автомагистраль | Прочие дороги | В населённом пункте |
|---|---|---|---|
| Легковые, лёгкий коммерческий | 140 | 110 | 60 |
| Грузовые ≤ 3,5 т, междугор. автобусы | 110 | 90 | 60 |
| Грузовые > 3,5 т, автобусы, с прицепом | 90 | 70 | 60 |
| Перевозка людей в кузове грузового | — | 60 | 60 |
| Буксировка | 50 | 50 | 50 |
| **Спец/тяжёлая на техдороге (внутр. стандарт)** | — | **50** | — |

### 3.3. Технический фундамент детекции превышений (уточнён по аудиту 22.06)

- **Макс. скорость** уже приходит из сводного API (`mv.maxSpeed`) — дешёвый предварительный сигнал.
- **Доля/факты превышений** из сводного API **не приходят** — досчитывать из **GPS-трека** против лимита (`track_clean.py` — чистка выбросов по физике).
- **Достоверность (R-INV-3):** превышение фиксируется **только по устойчивому сегменту** (≥ N последовательных правдоподобных точек / минимальная длительность), **не по одной точке**. Это убирает ложные срабатывания от GPS-выбросов.
- **Фильтр выбросов — недостаточно порога 200:** при лимитах зон 5–50 км/ч выброс 50–150 км/ч пройдёт `< MAX_PLAUSIBLE_SPEED_KMH(200)` и даст ложное «грубое превышение». Добавить **физический фильтр ускорения** (Δv/Δt > предел = выброс) + `satellitesCount ≥ 4` (в карьерах многолучёвость даёт ложные скорости и при ≥4 — отсюда сегмент).
- **Тип дороги обязателен (R-INV-1):** определить класс геозоны (Зоны 1–4 техдороги/полигоны vs 5/6 дороги общего пользования, [09 §1](09-kap-geozone-speed-standard.md)) → от него зависит, штраф КоАП это или дисциплинарка СТ КАП.
- **Две планки:** дисциплинарный флаг (СТ КАП: ≥6 км/ч) — мягче; денежная квалификация по КоАП — выше планка достоверности.
- Классификация диапазона: СТ КАП (≤3/3–5/≥6) для дисциплинарки; КоАП (10–20/20–40/40–60/60+) — только для дорог общего пользования ([03 §1](03-legal-framework-kz.md)).
- **Масштаб (R-арх):** трек по всем ~2000 ТС на каждом цикле нереализуем (1 GET/ТС, лимит ~150 req/мин). Тянуть трек **только для ТС-кандидатов** (где суточный `mv.maxSpeed` > мин. лимита зоны), отдельным **ночным батчем**, не в основном роллапе 8×/сутки. См. [SOLUTION §3.2](../SOLUTION.md).

## 4. Износ как отдельный KPI (вводная директора про КрАЗ/Урал)

Тяжёлая техника «выходит из строя» от превышения по пересечёнке — это не только штраф, но и **поломка/износ**. В движке:
- Связать превышения тяжёлой/спецтехники на технологических дорогах с **оценкой износа** ([03 §5.3](03-legal-framework-kz.md): доп. износ узлов / ресурс).
- Это усиливает экономическую рекомендацию: «превышение Х → не штраф (техдорога вне ПДД), а ускоренный износ на ~N ₸ + риск простоя».
