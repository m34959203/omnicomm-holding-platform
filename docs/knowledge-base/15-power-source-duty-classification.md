# Энергоноситель и режим работы: ДЭС, компрессоры, буровые — правила аналитики

> Вводные директора по развитию (аудио, 02.07.2026, передано Дмитрием). Дополняет
> [04 таксономию](04-vehicle-taxonomy-and-speed-policy.md) и [01 каталог датчиков](01-sensors-and-systems-catalog.md).
> Реализация маркеров — `vehicle_types.py` (`primary_metric`) + жёсткий фильтр в `api/fuel_norms.py`.

## 1. Главное правило — маркеры типа жёстко разделяют дашборды

Профиль ТС несёт **маркер метрики расхода**, и программа обязана фильтровать по нему,
не смешивая статистики (цитата: «по этим я эту статистику строю, по этим — ту, и между
собой не мешаю»):

| Маркер (`primary_metric`) | Кто | Правило |
|---|---|---|
| `l_per_mh` (моточасная) | ДЭС, компрессоры дизельные, трактора, бульдозеры, грейдеры, буровые, экскаваторы, краны | **НИКОГДА не попадает в дашборд л/100 км** |
| `l_per_100km` (дорожная) | легковые, автобусы, самосвалы, грузовые | **НИКОГДА не попадает в л/моточас** |
| `both` | УРБ, АГП, каротаж, КДМ (едут + работают агрегатом) | обе метрики, разбор по активности |
| `none` (электрическая) | **электрические компрессоры** («Компрессор (эл. …)» у Волковгеологии), эл. установки | **вне ВСЕЙ топливной аналитики** — ГСМ не потребляет; мониторить наработку/онлайн |

Маркеры типов Omnicomm тянуть в профиль ТС по мере появления (сейчас тип — из имени,
override > имя > эвристика, см. [аудит](../audit-vehicle-type-speed-limit.md)).

## 2. ДЭС (дизель-электростанции) — отдельная категория

- Обороты двигателя **постоянны** (напр. ~1200 об/мин под 50 Гц) → метрика «холостой
  ход / не холостой по оборотам» **неприменима** — ДЭС «всегда на холостых» с точки
  зрения обычной логики, и это ложный сигнал.
- Корректная метрика: **л/моточас** (при появлении данных выработки — л/кВт·ч).
- Тип `des` в `vehicle_types.py`; в карточке нет «топлива простоя».
- Сейчас ДЭС дополнительно отсечены вентилем `classify.is_transport=False` от денежных
  рейтингов ТС — это правильно (они не транспорт); их топливный мониторинг — отдельный
  контур, не смешивать с автопарком.

## 3. Компрессоры — электрические vs дизельные

- **Электрические** (в фиде Волковгеологии: «Компрессор (эл. Atlas XAXS600E)…») —
  ГСМ не потребляют. Тип `compressor_electric`, primary `none`: **не брать их моточасы
  в топливную аналитику вообще** (был баг: классифицировались как дизельные).
- **Дизельные** (Atlas Copco V900/XRVS336…) — тип `compressor`, только л/моточас.
- **КПД дизельного компрессора — по давлению магистрали, не по оборотам** (обороты —
  лишь косвенный признак):
  ```
  датчик давления магистрали (требовать от ДЗО оснащение):
      давление ≥ 10 бар → полезная работа (КПД+)
      двигатель работает, давление < 10 бар → холостое молочение (потери)
  пример: из 16 моточасов за сутки под давлением только 2 ч → КПД 2/16,
          14 ч — потери (топливо ₸)
  ```
  Статус: датчиков давления в фиде `projectkap` нет (`uniDataList` пуст) →
  **фича «при дооснащении»**; модель метрики готова, ждёт данных.

## 4. Буровые — КПД по датчику бурения

- У буровых уже снимается **«бурит / не бурит»** доп. датчиком на станке (контур ДЗО).
- Правило КПД: **бурит → КПД+; не бурит → КПД−**, даже если в этот момент насосы
  качают, допоборудование работает и «станок работает» — процесс бурения остановлен
  = потери.
- Статус: сигнал датчика бурения в наш фид Omnicomm не приходит → интеграция при
  доступе к данным датчика; до этого л/моточас без разбивки на бурение/простой.
