# Возможности Omnicomm Online (по руководству пользователя, ред. 07.02.2026, 206 стр.)

> Разбор `download/omnicomm_online-manual-ru.pdf`. Цель: понять, что САМА система уже умеет — чтобы не переизобретать и знать, какие данные/отчёты/события доступны для забора через API.
> Источник извлечён 2026-06-22.

## 0. Ключевой вывод
Omnicomm Online — полнофункциональная отчётно-мониторинговая система с **~50 встроенными отчётами**, своей детекцией событий, геозон, нарушений, ТО и водителей. Наша holding-надстройка **НЕ переизобретает** расчёт топлива/нарушений/моточасов — всё считается на стороне Omnicomm. Наша ценность — **агрегация по 23 ДЗО, кросс-объектные рейтинги, KPI-дашборды холдинга, нормализация и хранение трендов**. Терминология ниже — «язык заказчика».

**Гоча:** REST API и форматы интеграции в этом руководстве НЕ описаны — они в отдельном «Руководстве по администрированию» (на него ~15 ссылок). Здесь только UI-отчёты и выгрузки. Подтверждает наш подход тянуть отчёты через REST (POST).

## 1. Состав системы
- **Роли:** Пользователь (строит отчёты, профили, регистрация водителей, уведомления) · Дилер (создаёт пользователей/права, импорт-экспорт) · Администратор (сотрудник Omnicomm).
- **Аутентификация:** двухэтапная (логин/пароль + SMS). Адрес `online.omnicomm.ru`.
- **Объекты:** ТС (**Машины** и **Топливозаправщики** — разные типы!), Водители, Геозоны, Маршруты.
- **Рабочие столы:** «Руководителя» (сводка по всем ТС, обновление 30 мин) и пользовательский (до 10–20 отчётов). На базе «Топливный баланс/Ведомость топливозаправщика/Детализация нарушений/Рейтинг водителей» — строить нельзя.
- **Дерево объектов:** фильтры активности (online/offline, «не поступали более», «данные отсутствуют»).
- **Доступ по ссылке без авторизации:** только Местоположение, Трек, SafeDrive (Детализация нарушений, Рейтинг водителей) + рабочие столы из них — через техподдержку, со сроком действия. **← потенциальный канал встраивания в наш холдинг-портал.**
- **Прочее:** Контроль ТО, Видеостриминг (ОКО, до 16 камер), Уведомления, Управление выходом терминала, Техподдержка.

## 2. Отчёты (~50, по группам)
- **Топливные:** Заправки и сливы (LLS, сравнение с топливными картами: пороги 15 мин / 3%) · Объём топлива (сырые/сглаженные) · Объём топлива (моточасы) · Выдача топлива (топливозаправщик) · Выдачи/заливы/сливы из цистерны (хищения) · Ведомость топливозаправщика (сопоставление выдач↔заправок, RFID/iButton) · **Топливный баланс** (остаток→поступило/расход/сливы/потери→конечный) · MultiTank (несколько ёмкостей).
- **Картографические:** Трек (раскраска по превышению/зажиганию/доп.оборудованию) · Местоположение (real-time, ≥4 спутника) · Стоянки · Скорость (зелёный ≤ лимит, красный >) · Движение / Распределение движения.
- **Статистические (ключевые для KPI):** **Статистика** (десятки параметров: пробег, ср/макс скорость, **пробег с превышением**, моточасы, расход осн/доп, расход на 100 км и моточас, норма, отклонение %, перерасход, **масса топлива кг, выбросы СО2**, счётчики CAN, до ТО, iQFreeze, TPMS) · **Сводный отчёт** (группировка по ТС/группам/дате) · Посменный (до 4 смен).
- **Работа:** Обороты двигателя (холостые/нормальные/предельные) · Напряжение бортсети · Работа доп.оборудования · Рефрижератор (iQFreeze) · Давление в шинах (TPMS) · Нагрузка/Работа за период.
- **События и нарушения:** Нарушения · События · Пользовательские события (.xlsx) · **SafeDrive: Детализация нарушений** · **SafeDrive: Рейтинг водителей**.
- **Другие:** Текущее состояние · Посещение геозон (вход/выход, пробег с превышением, для агро — га/час, л/га) · Журнал (сырьё) · Регистрация водителей · Мультимедиа · Рейсы · Работа/Рейтинги группы.

## 3. Геозоны
- Фигуры: многоугольник, окружность (радиус+центр), линия (10–1000 м), карта (растр).
- Параметры: «Использовать вместо адреса» · **«Отслеживать нарушение скоростного режима»** · **«Разрешённая скорость, км/ч» — лимит, привязанный К ГЕОЗОНЕ** → события «Начало/Окончание превышения порога скорости в геозоне». **← отдельный механизм от лимита в профиле ТС.**
- **Динамическая геозона** (движущаяся, центр = базовое ТС): радиус 25 м, срабатывание 60 с, актуальность 180 с.

## 4. Скоростной режим / нарушения
**Два уровня лимита:** (1) глобальный в профиле ТС → «Превышение разрешённой скорости»; (2) на геозону → «Нарушение скоростного режима в геозоне». **«Порог длительности нарушения»** — сколько можно превышать без фиксации.
**Типы нарушений:** Топливо (Слив, Сбой LLS) · Движение (Стоянка, Остановка, Ускорение выше допустимого, Превышение скорости, Превышение в геозоне) · Двигатель (Превышение оборотов) · Прочее (некорректная регистрация водителя, вскрытие, тревожная кнопка) · Доп.оборудование · Маршруты (отклонение, контрольные точки, выход за границы) · **SafeDrive** (фары/ремни/ускорения/скорость входа в поворот/**Нарушение ПДД. Превышение**/холостой ход/непрогретый-перегретый двигатель/обороты/нет GPS). Пороги — в профиле ТС («Плохие привычки», в админке).

## 5. Уведомления / события
- **События** — полный поток (Топливо, Движение, Двигатель, Вход-Выход геозоны/динамической, iQFreeze, TPMS, SafeDrive, OBDII Check Engine, вскрытие, резервное питание).
- **Уведомления (алёрты):** до 100, настраиваются на событие + ТС + опц. «В геозоне», метки «важное»/«служебное», быстрый переход в Трек/Объём топлива.
- **Пользовательские события** — по параметрам ТС, экспорт .xlsx.
- **ТО:** по дате/пробегу/моточасам, «Ожидается ТО» → уведомление.

## 6. Датчики (из отчёта «Журнал»)
LLS до 6 (уровень, температура топлива, автоподстройка LLS 5/Ex 5) · GPS (≥4 спутника, **детект глушения GPS/GSM**) · Зажигание, напряжение, резервный АКБ · Обороты, скорость импульсная, **ускорение (акселерометр X/Y/Z)** · **CAN** (тормоз, педаль %, давление масла, температуры, расход/пробег сут/общий, моточасы, нагрузка на ось, двери, ремни, до ТО) · OBDII (VIN, Check Engine) · **4 универсальных входа УВ1–4** (аналог/импульс/потенциал) · iQFreeze (рефрижератор: ThermoKing/Carrier/Zanotti) · **TPMS** (давление/температура/батарея по колесу) · i-wire до 8 темп. · GenComm (генераторы) · Видео ОКО/OKO Light · Доп.ДУТ (ПМП-201, Струна+) · вскрытие, тревожная кнопка. Настройка — Omnicomm Configurator.

## 7. Водители
- Регистрация на ТС: вручную или авто по **iButton / RFID**.
- Незарегистрированный/уволенный ключ → «Некорректная регистрация водителя».
- **Все статистические и топливные параметры строятся «по водителю»** (за период регистрации). SafeDrive-рейтинг — по водителям. → технический фундамент «досье водителя» из [05](05-recommendation-engine.md).

## 8. API / экспорт
- REST API — в отдельном «Руководстве по администрированию» (запросить).
- **Экспорт из UI:** отчёты → **Excel (.xls/.xlsx)** + печать.
- **Доступ по ссылке** (встраиваемые отчёты без авторизации) — через техподдержку.
- Импорт/экспорт объектов — пользователю/дилеру.

## 9. Единицы и гочи (для интеграции)
- Топливо — литры (или галлоны по настройке сервера); масса — кг. Скорость — км/ч; пробег — км (CAN 0.1); моточасы — час:мин.
- **Движение** = скорость >2 км/ч И зажигание вкл. **Работа двигателя** = обороты >10 об/мин.
- **«Машина» vs «Топливозаправщик»** — разные типы с разными отчётами.
- **Данные пересчитываются** на стороне Omnicomm → отчёт может быть частичным во время пересчёта.
- **Сырые vs сглаженные** данные топлива — два графика.
- **GPS валиден только при ≥4 спутниках**; есть детект глушения.
- **Двойной лимит скорости** (профиль ТС + геозона) — учитывать оба.
- **Выбросы СО2** (2392 бензин / 2640 дизель × расход) — готовый ESG-показатель для КАП.

**Что Omnicomm уже умеет (НЕ переизобретать):** расход/нормы/отклонения, заправки/сливы/выдачи, скоростные нарушения, SafeDrive-рейтинг, моточасы, посещение геозон, посменка, ТО, топливный баланс, агро (га), рефрижератор, шины, СО2.
**Наша ценность:** кросс-ДЗО агрегация (23 ДЗО), сводные KPI холдинга, бенчмаркинг ДЗО, единый дашборд без Power BI, долговременное хранение/тренды.
