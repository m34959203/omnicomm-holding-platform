# Каталог датчиков и бортовых систем мониторинга транспорта

> Справочник телематического оборудования для управления автопарком. Центр — экосистема **Omnicomm** (терминалы, ДУТ LLS, видеотелематика OKO), с покрытием всего рынка и аналогов.
> Обозначение **(уточнить)** — модель/цифру стоит сверить с актуальной спецификацией перед коммерческим предложением.
> Источник: ресёрч-сборка 2026-06-22 (omnicomm.ru, doc.omnicomm.ru, дилерские каталоги — ссылки в конце).

---

## 1. GPS/ГЛОНАСС-терминалы (бортовые трекеры)

Сердце системы: собирает данные со всех датчиков, определяет местоположение и передаёт телеметрию на сервер (Omnicomm Online / online.omnicomm.ru).

### 1.1. Omnicomm Light — базовый терминал
- **Что измеряет / принцип:** Координаты по ГЛОНАСС/GPS, скорость, пробег; встроенный акселерометр (стиль вождения, удары). Встроенные GSM и ГЛОНАСС/GPS антенны.
- **Какие задачи закрывает:** Базовый мониторинг местоположения, контроль маршрута и пробега, фиксация простоев. Минимальный бюджет на ТС.
- **Какие данные даёт:** Трек (координаты/скорость), пробег, моточасы по зажиганию, события геозон, базовые события вождения. 2 универсальных входа + вход оборотов двигателя, датчик вскрытия (tamper).
- **Когда рекомендовать:** Лёгкий коммерческий транспорт, такси, развозка, большой парк с ограниченным бюджетом, где не нужен «тяжёлый» контроль топлива через RS-485.

### 1.2. Omnicomm Optim — массовый терминал
- **Принцип:** То же ядро + расширенный набор интерфейсов для подключения внешних датчиков (RS-232/RS-485, 1-Wire, входы/выходы).
- **Задачи:** Полный контроль топлива (через ДУТ по RS-485), идентификация водителя, контроль дверей/температуры — оптимальная цена для массовой установки.
- **Данные:** Трек, топливо (уровень/расход/заправки/сливы при ДУТ), CAN-данные (FMS/J1939), температура, события датчиков.
- **Когда рекомендовать:** Основная рабочая лошадка парка — грузовики, автобусы, спецтехника с топливным контролем и идентификацией.

### 1.3. Omnicomm Profi / Profi Wi-Fi — защищённый терминал
- **Принцип:** Как Optim, но максимальная надёжность: два канала идентификации SIM (карта + чип) для резервирования; Profi Wi-Fi выгружает накопленные данные по Wi-Fi (в гараже).
- **Задачи:** Тяжёлые/«северные» условия, зоны с плохим GSM (карьеры, удалённые трассы), гарантированная сохранность данных.
- **Когда рекомендовать:** Нефтегаз, горная/строительная техника, Крайний Север, магистраль. **← профиль КАП (карьеры, рудники).**

### 1.4. Терминалы 3.0 / АСН (аппаратура спутниковой навигации)
- Сертифицированные решения под госрегуляторов (тахографические/навигационные нормы), интеграция с ЭРА-ГЛОНАСС **(уточнить актуальный сертификационный статус для РК)**.
- **Когда рекомендовать:** Парки, обязанные к сертифицированному оснащению (опасные грузы, пассажирские перевозки).

**Аналоги (сторонние трекеры):** Teltonika (FMB/FMC/FMU), Galileosky, Naviset, АвтоГРАФ (ТехноКом), СКАУТ, Gurtam-совместимые. Все принимают ДУТ по LLS-протоколу → датчики Omnicomm совместимы и со сторонними трекерами.

## 2. Топливный контроль

Самый частый драйвер внедрения (экономия 15–30 % топлива). Три подхода: ёмкостный ДУТ, штатный CAN/FMS, проточный расходомер.

### 2.1. ДУТ — датчик уровня топлива (Omnicomm LLS)
- **Принцип:** Ёмкостный измеритель уровня в баке (измерительная трубка погружается в топливо). Технология **Fuelscan** — автоподстройка под тип/температуру топлива, точность «до 99,5 %», диапазон −60…+85 °C.
- **Какие задачи закрывает:** KPI «топливо» — выявление **сливов** (резкое падение вне заправки), контроль **заправок** (факт vs чек), реальный расход, нормирование, борьба с приписками. Главный инструмент окупаемости.
- **Какие данные даёт:** Уровень (объём/%), графики заправок/сливов с объёмом+местом+временем, расход на 100 км и на моточас, остаток на начало/конец смены.
- **Модели:** Omnicomm **LLS 5** / **LLS-Ex 5** (взрывозащита, IP69K); **LLS 30160** (RS-232/RS-485). Сторонние: Эскорт ТД, Технотон DUT-E.
- **Когда рекомендовать:** Любой ТС с риском слива, где CAN не даёт уровня (старые грузовики, спецтехника, генераторы). **Ex-версия — топливозаправщики, опасные зоны.**

### 2.2. CAN/FMS — штатный расход с шины автомобиля
- **Принцип:** Чтение заводских данных бортового компьютера по **FMS** / **J1939** через CAN-интерфейс терминала или бесконтактный считыватель.
- **Задачи:** Расход без врезки в бак (быстрый монтаж, не нарушает гарантию), уровень по штату, мгновенный/средний расход.
- **Данные:** Полный/текущий расход, уровень, пробег по одометру, обороты, скорость, нагрузка.
- **Модели:** Встроенный CAN Optim/Profi; внешний бесконтактный **CAN-LOG**, **Sukel**.
- **Когда рекомендовать:** Современные грузовики/автобусы с FMS. Часто комбинируют с ДУТ для перекрёстной проверки.
- **Типы подключения к CAN** (со слов Антона, 22.06): «крокодилы» (бесконтактный считыватель-накладка, рекомендация вендора, без врезки), **CAN-LOG** (бесконтактный считыватель), либо прямое подключение. Способ влияет на надёжность CAN-данных — учитывать при детекции сбоев ([10](10-sensor-health-detection.md)).

### 2.3. ДРТ — проточный расходомер топлива
- **Принцип:** Измеряет объём топлива через магистраль (диф-схема: подача минус «обратка»).
- **Задачи:** Точный расход там, где нет ровного бака или нужен расход «на работу» (генераторы, ДГУ, судовые/стационарные двигатели).
- **Данные:** Мгновенный и накопленный расход (л/ч, л) независимо от уровня в баке.
- **Модели:** Технотон DFM, Eurosens Direct/Delta (сторонние, частотный/RS-485 вход) **(в линейке Omnicomm обычно отсутствуют — уточнить)**.

## 3. Двигатель и CAN-шина

### 3.1. FMS-адаптер / CAN-интерфейс
- **Данные:** Обороты (RPM), температура ОЖ, нагрузка (%), положение педали, давление масла, уровень AdBlue, общий пробег/расход, моточасы, **коды ошибок DTC**.
- **Задачи:** Глубокая диагностика, контроль режима двигателя, предиктивное ТО.
- **Когда рекомендовать:** Современная техника, где нужна диагностика и контроль ТО.

### 3.2. Датчик оборотов двигателя (тахометр)
- **Принцип:** Импульсный вход (W-клемма генератора, форсунка, индуктивный датчик маховика).
- **Задачи:** Контроль работы двигателя без CAN, выявление «прогрева вхолостую», разделение «движение / холостой ход / стоянка».
- **Данные:** Обороты, моточасы по факту работы, доля холостого хода.
- **Когда рекомендовать:** Старая техника без CAN, спецтехника, где важны моточасы. **← буровые/спецтехника КАП.**

### 3.3. Датчик зажигания / контроль работы двигателя
- Дискретный вход от замка/бортсети → состояние «зажигание вкл/выкл», моточасы, контроль несанкционированного запуска.

### 3.4. Контроль AdBlue / SCR
- Уровень и расход реагента по CAN или отдельным ДУТ → контроль расхода, выявление «обхода» нейтрализации, защита от аварийного режима двигателя. Магистральные тягачи Евро-5/6.

## 4. Спецтехника

### 4.1. Датчик работы оборудования (универсальный вход / PTO)
- **Принцип:** Дискретный/аналоговый вход к цепи включения гидравлики/насоса/шнека/миксера или датчик давления в гидросистеме.
- **Задачи:** KPI «КПД спецтехники» — учёт реальной полезной работы (а не «двигатель работал»), разделение «работа/переезд/простой», нормирование расхода на единицу работы.
- **Данные:** Состояние «оборудование работает», моточасы оборудования, привязка к месту/времени.
- **Когда рекомендовать:** Экскаваторы, краны, бетоносмесители, дорожная/коммунальная техника.

### 4.2. Датчик угла/наклона кузова (самосвал)
- Геркон/инклинометр → события «кузов поднят/опущен», количество разгрузок, место/время. Контроль несанкционированных разгрузок «налево», учёт рейсов. **← самосвалы в карьерах КАП (оплата по рейсам).**

### 4.3. Датчик веса / нагрузки на ось
- Тензодатчики/давление в пневмоподвеске/CAN → нагрузка на ось/масса (т), события перегруза, масса по рейсам. Контроль перегруза (штрафы, износ), учёт массы груза.

### 4.4. Обороты вала отбора мощности (ВОМ)
- Импульсный вход → контроль режима навесного оборудования (сельхоз/коммунальная техника), привязка к выработке.

## 5. Груз и прицеп

### 5.1. Датчик открытия дверей
- Геркон/концевик → события «дверь открыта/закрыта» с координатой/временем. Контроль несанкционированного доступа, фиксация разгрузок, сохранность.

### 5.2. Температура рефрижератора
- Цифровые датчики **1-Wire** (до 8 шт на Optim/Profi) → температура по зонам (°C), графики, выход за пороги. KPI «холодовая цепь», претензионная защита (фарма/продукты).

### 5.3. Контроль сцепки прицепа
- Концевик ССУ или BLE-маяк прицепа → события сцепки/расцепки, ID прицепа. Контроль угона/расцепки, учёт «тягач-прицеп». **← «шаланды/полуприцепы» из вводных директора.**

## 6. Идентификация водителя

### 6.1. iButton / Dallas (1-Wire)
- Считыватель 1-Wire, ключ-«таблетка» при начале смены → ID водителя, время смены, **привязка нарушений/расхода к ФИО**. Комплект **Omnicomm KI-4**. В связке с иммобилайзером = разблокировка ТС.

### 6.2. RFID-карты
- RFID-ридер (125 кГц / 13,56 МГц) → то же, что iButton, но удобнее (единая карта-пропуск СКУД).

> **Критично для движка рекомендаций:** без идентификации водителя нарушение привязывается к ТС, а не к человеку. Директор требует «досье водителя» (год рождения и т.д.) — идентификация = технический фундамент этого.

---

## 7. Безопасность и тревога

### 7.1. Тревожная кнопка SOS
- Дискретный вход → приоритетное событие «тревога» с координатой. Безопасность водителя (нападение, ЧП), пассажирские перевозки, инкассация.

### 7.2. Иммобилайзер / блокировка двигателя
- Релейный выход размыкает цепь (стартер/насос) по команде с сервера, с защитой «только на безопасной скорости». Противоугон, дистанционная остановка.

### 7.3. Датчик вскрытия (tamper)
- Встроенный геркон + контроль пропадания питания → событие «вскрытие/отключение». Антисаботаж. По умолчанию для всех ТС.

---

## 8. Шины — TPMS
- Беспроводные датчики на колёсах (давление+температура по RF/BLE) → события «низкое давление / перегрев / быстрая потеря». KPI «топливо+безопасность+износ»: предотвращение разрыва шины, экономия топлива, снижение износа резины. Магистраль, автобусы, дорогая резина.

---

## 9. Видео и поведение водителя (обзорно — детально см. [02-driver-safety-systems.md](02-driver-safety-systems.md))

> ⚠️ **Ограничение интеграции (вводная директора 22.06):** видеотерминалы на технике КАП — **не производства Omnicomm и в Omnicomm напрямую данные не передают** (своё ПО, отдельный контур). Значит видео/ADAS/DSM **не приходят через Omnicomm API** — это отдельная интеграция. Рекомендация «дооснастить DSM/ADAS» = продажа отдельной системы, а не включение фида. Подробнее — [holding-product-requirements.md §3](../holding-product-requirements.md).

| Система | Что | KPI |
|---|---|---|
| **MDVR** (Omnicomm OKO / OKO Light) | Видеотерминал + до 4 камер, видео/фото синхронно с телеметрией | Разбор ДТП, контроль груза/разгрузки, претензии |
| **Камеры** (дорога/салон/360°) | Слепые зоны, контроль кабины/маневрирования | Безопасность маневрирования |
| **ADAS** | Фронт-камера + ИИ: сход с полосы (LDW), риск столкновения (FCW), дистанция, знаки | KPI «безопасность» снаружи |
| **DSM** | Кабинная ИК-камера + ИИ: усталость, отвлечение, телефон, курение, ремень | KPI «безопасность» внутри |
