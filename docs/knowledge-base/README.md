# База знаний платформы — «мозг технического специалиста»

> Назначение: дать платформе **знания технического специалиста по телематике** — какие датчики и системы бывают, что каждый закрывает, какие данные даёт и **когда что рекомендовать клиенту**. Это одновременно (1) фундамент движка рекомендаций, (2) консультативно-продажный слой для интегратора, (3) справочник для подбора решения под потребности ДЗО.
> Старт: 2026-06-22 по запросу Дмитрия + вводным директора по развитию Omnicomm.

## Логика: как платформа «думает как техспециалист»

```
Потребность/боль клиента ──► [01/02] какой датчик/система закрывает ──► подбор решения
Телеметрия (трек, топливо, обороты) ──► [04] лимит по геозоне×типу ТС ──► детекция нарушений
Нарушение ──► [03] статья КоАП + штраф + ущерб ──► [05] конкретная рекомендация + действие
Систематика ──► [05] лестница эскалации ──► дооснащение ADAS/DSM = инструмент продаж
```

Пример «потребность → решение»: *«воруют топливо»* → ДУТ Omnicomm LLS ([01 §2.1](01-sensors-and-systems-catalog.md)); *«спецтехника простаивает под видом работы»* → датчик PTO/гидравлики ([01 §4.1](01-sensors-and-systems-catalog.md)); *«водители превышают и бьют технику»* → детекция по треку ([04](04-vehicle-taxonomy-and-speed-policy.md)) + при систематике DSM/ADAS ([02](02-driver-safety-systems.md)).

## Директивы директора по развитию Omnicomm (аудио 22.06.2026) — закреплено

1. **Рекомендации — на букве закона, не «ИИ-вода».** Структура: нарушение → статья ПДД/КоАП → количество/частота → сумма штрафов → вероятный ущерб → действие (беседа с водителем). → [05](05-recommendation-engine.md). *(Антипаттерн — текущий «ИИ-ПОМОЩНИК» в интерфейсе, выдающий общий план без привязки к статье/сумме.)*
2. **Досье водителя** (год рождения, стаж и т.д.) — подцеплять к рекомендации; требует идентификации водителя. → [05 §3](05-recommendation-engine.md).
3. **Эскалация к железу = инструмент продаж:** систематические нарушения за месяц → рекомендовать видеоконтроль + DSM + ADAS. → [05 §4](05-recommendation-engine.md), [02 §7](02-driver-safety-systems.md).
4. **Таксономия ТС:** опасные грузы, шаланды/полуприцепы, пассажирские, автобусы, тяжёлая/спец (КрАЗ/Урал — «выходят из строя от превышения по пересечённой местности»). → [04](04-vehicle-taxonomy-and-speed-policy.md).
5. **Скоростной лимит 50 км/ч** на пересечённой/технологической местности (как у ИНКАЯ). Юр-нюанс: это внутренний стандарт холдинга (город по ПДД = 60). → [04 §3](04-vehicle-taxonomy-and-speed-policy.md), [03 §4](03-legal-framework-kz.md).

**Согласовать с людьми (поручения директора):**
- 🔴 **Данияр** — список опасных грузов холдинга (под ADR-настройку).
- 🟠 **Антон** — обкатка функционала/таксономии.
- 🟠 **Дима** — переслать вводную про ADAS/DSM как инструмент продаж.

## Контекст интерфейса (скриншоты 22.06)
- **Дашборд:** таблица нарушений скоростного режима в реальном времени (дата/время, ТС, скорость, лимит, превышение, длительность, локация) + панель «ИИ-ПОМОЩНИК».
- **Карточка ТС:** карта + позиция, инфо-окно (скорость 70.4, бортсеть, пробег, геозона «Дорога до Рудник Инкай 80 км/ч»), график скорости во времени.
- Геозона уже несёт лимит → подтверждает модель `geozone_limit` ([04](04-vehicle-taxonomy-and-speed-policy.md)). «ИИ-ПОМОЩНИК» в текущем виде = тот самый антипаттерн, который заменяет движок [05](05-recommendation-engine.md).
