## 0. Красные линии (инварианты) — обязательны, нарушение = блокер релиза

> Внесено по итогам экспертного аудита 22.06 ([REVIEW](../REVIEW-2026-06-22.md)). Движок ОБЯЗАН исполнять правовые и качественные оговорки, а не только декларировать их.

- **R-INV-1 (право дороги).** Статья КоАП РК + сумма в ₸ — **только** для геозон типа «дорога общего пользования» (Зоны ПДД 5/6: магистрали `M-/A-/P-`, населённые пункты). На технологических дорогах / ГТП / полигонах / промплощадках (Зоны 1–4 СТ КАП) — **только дисциплинарное отклонение по СТ КАП**, без статьи и без суммы. КоАП налагается органами УДП на дорогах общего пользования, не работодателем на ведомственной дороге.
- **R-INV-2 (нет фиктивных сумм).** Запрещена персональная «суммарная задолженность по штрафам» к водителю (7 точек трека ≠ 7 постановлений). Деньги — только агрегированный обезличенный риск-индикатор с оговоркой «оценочно, не начисленный штраф».
- **R-INV-3 (достоверность скорости).** Превышение — только по устойчивому сегменту трека (≥N правдоподобных точек + фильтр ускорения), не по одной GPS-точке; для денежной квалификации планка достоверности выше, чем для дисциплинарного флага. См. [04 §3.3](04-vehicle-taxonomy-and-speed-policy.md).
- **R-INV-4 (вентиль качества).** ТС с непроверенным/«грубо»-валидным ДУТ — не в денежные рейтинги без confidence-флага ([10](10-sensor-health-detection.md)).
- **R-INV-5 (verified seed).** Квалификация — только по отвалидированному справочнику геозон (без дублей-конфликтов/опечаток/чужих имён, [09 §5](09-kap-geozone-speed-standard.md)); геозоны без чистого имя-матча к Omnicomm деньгами не квалифицируются.
- **R-INV-6 (нет авто-кадровых решений).** Платформа информирует, не предписывает депремирование/приказ; формулировки — «к рассмотрению руководителем ДЗО».
- **R-INV-7 (ПДн).** Досье водителя (§3) — только при правовом основании (согласие/локальный акт), минимизации полей, без авто-негативных решений по возрасту.
- **R-INV-8 (сверенный закон).** Суммы — только по статьям со статусом «сверено с adilet» + `koap_revision_date` в `config.py`; несверённые составы не показывать с суммой.

## 1. Принцип: НЕ «обобщённая ИИ-вода»

Директор прямо: *«Рекомендации, которые им даёт их искусственный интеллект, считаю полной хернёй. Наши рекомендации должны в первую очередь опираться на букву закона»*.

Антипаттерн (то, что сейчас выдаёт «ИИ-ПОМОЩНИК» на скриншоте интерфейса): общий план «определить типы нарушителей → создать систему мониторинга → обучение водителей → ТО → уведомления → автоштрафы». Это вода — не привязана к конкретному ТС, статье, сумме, действию.

**Наш стандарт рекомендации = конкретика + закон + деньги + действие:**

```
Нарушение (что, где, когда, ТС, водитель)
  → Статья КоАП РК + квалификация (диапазон превышения)
  → Количество и частота (за период, динамика)
  → Сумма штрафов (МРП × 4 325 ₸, с учётом повторности)
  → Вероятный ущерб (ДТП-риск + износ + простой)
  → Конкретное действие (беседа с водителем / приказ / дооснащение)
```

## 2. Структура карточки рекомендации (шаблон)

Каждая рекомендация платформы должна содержать поля:

| Поле | Источник | Пример |
|---|---|---|
| **Объект** | дерево + идентификация | ТС «TOYOTA LC Prado 250 709AL11», водитель (если есть ID) |
| **Нарушение** | трек × geozone_limit ([04](04-vehicle-taxonomy-and-speed-policy.md)) | превышение на 35 км/ч в зоне «Рудник Инкай, 80 км/ч» |
| **Квалификация** | [03 §1](03-legal-framework-kz.md) | ст. 592 ч. 2 КоАП РК (20–40 км/ч) |
| **Частота** | история нарушений | 7 раз за месяц, рост к прошлому |
| **Штраф** | МРП × ₸ + повторность | 10 МРП = 43 250 ₸ за случай; при повторе за год ч. 4 = 30 МРП = 129 750 ₸ |
| **Ущерб** | [03 §5](03-legal-framework-kz.md) | риск ДТП ↑ (Nilsson) + износ тормозов/трансмиссии ≈ N ₸ + простой |
| **Действие** | лестница эскалации (§4) | провести беседу с водителем по факту систематических превышений |

## 3. Досье водителя (вводная директора)

*«Если бы мы знали, что за водитель, какого он года рождения, можно ещё и подцепить что-то туда»*.

Рекомендация персонализируется при наличии **досье водителя**:
- **Технический фундамент:** идентификация водителя (iButton/RFID — [01 §6](01-sensors-and-systems-catalog.md)). Без неё нарушение привязано к ТС, не к человеку.
- **Поля досье:** ФИО, год рождения, стаж, категория прав, допуск ДОПОГ (срок!), история нарушений и бесед, балл вождения (driver score — [02 §4](02-driver-safety-systems.md)).
- **Применение:** молодой/малоопытный + систематические превышения → усиленная рекомендация (наставничество, ограничение на тяжёлую технику); истекающий ДОПОГ у водителя опасных грузов → блокирующая рекомендация.

## 4. Лестница эскалации (от беседы до «железа» = инструмент продаж)

Вводная директора: *«при систематических нарушениях в течение следующего месяца — рекомендация оснастить систему видеоконтроля, ADAS, DSM… Вот тебе дополнительный инструмент продаж»*.

| Ступень | Триггер | Рекомендация | Бизнес-эффект |
|---|---|---|---|
| 1. Разовое | единичное превышение | информировать, зафиксировать | дисциплина |
| 2. Беседа | несколько за период | провести беседу с водителем (по статье + сумме) | управленческое действие |
| 3. Приказ/депремирование | систематические, тяжёлые (40+ км/ч) | дисциплинарная мера, привязка к KPI водителя | снижение риска |
| 4. **Дооснащение (upsell)** | **систематические в течение месяца** | оснастить ТС **видеоконтролем + DSM + ADAS** ([02](02-driver-safety-systems.md)) | **продажа оборудования** |
| 5. Программа | парк/ДЗО с высоким риском | программа безопасности по ДЗО, пилот с ROI | recurring + масштаб |

Ступень 4 — это и безопасность, и **воронка продаж интегратора**: нарушения, выявленные софтом, автоматически обосновывают продажу железа. Детальная воронка upsell — в [02 §7](02-driver-safety-systems.md).

> Что НЕ дадим без железа (честная граница): «доля превышений» — досчитываем из трека (есть); аварийность/резкие манёвры/усталость/телефон — **требуют акселерометра/DSM** ([02](02-driver-safety-systems.md)). На сводном API копии КАП `uniDataList` (надстроечные датчики) не пришёл — поведенческие метрики идут только при дооснащении. Это и есть естественный повод для ступени 4.

## 5. Пример «до / после»

**❌ Конкурент (ИИ-вода):** «Рекомендуется определить типы нарушителей, создать систему мониторинга, провести обучение водителей и внедрить систему автоматического взыскания штрафов.»

**✅ Наша рекомендация (с учётом инвариантов §0):**
> ТС «TOYOTA LC Prado 250 709AL11» — **устойчивое превышение** на сегменте в зоне «Дорога до Рудник Инкай (лимит 80 км/ч)», макс +35 км/ч, **7 эпизодов за июнь** (по устойчивым сегментам, не отдельным точкам — R-INV-3).
> — Если участок — **дорога общего пользования** (R-INV-1): справочно ст. 592 ч. 2 КоАП РК (20–40 км/ч), ставка 43 250 ₸ за случай **при условии фиксации органом УДП** (не начисленный штраф, R-INV-2).
> — Если участок — **ведомственная/технологическая дорога рудника** (вероятно для подъезда к Инкаю): это **дисциплинарное отклонение по СТ КАП**, без статьи КоАП и без суммы в ₸ (R-INV-1).
> **Риск-индикатор по ТС/ДЗО** (обезличенно, оценочно): частота превышений ↑, рост риска ДТП (Nilsson) и износа — **отдельно от правовой части, не суммировать** (R-INV-2).
> **Действие (к рассмотрению руководителем ДЗО, R-INV-6):** рассмотреть беседу с водителем; при сохранении динамики — оценить дооснащение DSM (контроль отвлечения/усталости).

> ⚠️ Тип дороги (общего пользования vs технологическая) определяется классом геозоны из СТ КАП ([09 §1](09-kap-geozone-speed-standard.md)) и обязателен на входе квалификации (R-INV-1).

## 6. Привязка к коду

- Базис — существующий `alerts.py` (авто-сигналы: перерасход/простой/тёмные ТС + добавленный скоростной режим). Движок рекомендаций надстраивается: сигнал → квалификация по [03](03-legal-framework-kz.md) → сумма → действие.
- Превышения — `track_clean.py` + новый расчёт `geozone_limit` ([04 §3.3](04-vehicle-taxonomy-and-speed-policy.md)).
- Справочник статей/штрафов — зашить из [03](03-legal-framework-kz.md) в `config.py` (как `OMNICOMM_ERRORS`), не хардкодить по месту. МРП 2026 = 4 325 ₸ — вынести в конфиг (меняется ежегодно).
- Бизнес-инвариант проекта сохраняется: формулировки без обвинений, «требует проверки»; здесь же — строго по статье, без домыслов о вине.
