# Стандарт скоростных режимов и геозоны Казатомпром (СТ КАП)

> Разбор файла `download/драфт с геозонами_исправлено.pdf` (27 стр.) — это **СТАНДАРТ АО НАК «Казатомпром»** «Организационная система по обеспечению безопасной эксплуатации системы GPS АТС и видеоаналитики водителей» (ССБТ) + плоский экспорт «Список геозон» из Omnicomm.
> **Прямой вход для модели `geozone_limit(геозона, тип_ТС) → макс_скорость`** ([04](04-vehicle-taxonomy-and-speed-policy.md)). Имя геозоны совпадает с интерфейсом Omnicomm (подтверждение: «Дорога до Рудник Инкай 80 км/ч»).
> Источник извлечён 2026-06-22.

## 0. Две части документа
- **Стр. 1–16** — нормативный текст СТ КАП (термины, мониторинг на базе **Omnicomm Online**, классификация ТС и зон, скоростная матрица, санкции, требования к подрядчикам).
- **Стр. 16–27** — «Список геозон»: экспорт из системы, сгруппирован по ДЗО, формат «№ → имя геозоны → лимит км/ч».

## 1. ДВА механизма лимита (критично для движка)

### Механизм А — Зонная матрица `f(зона риска, категория ТС)` (раздел 7)

**6 категорий территорий по риску (п. 7.5):**
| Зона | Описание |
|---|---|
| Зона 1 | Пешеходная/складская: постоянное присутствие людей, ПРР |
| Зона 2 | Производственная: дороги внутри промплощадки, стоянки |
| Зона 3 | Внутренние подъездные дороги между объектами |
| Зона 4 | **Полигон, технологические дороги** |
| Зона 5 | Населённые пункты |
| Зона 6 | Вне НП, трассы и автомагистрали общего пользования |

**Таблица 1 — предельные скорости, км/ч (п. 7.6):**
| Категория ТС | Зона 1 | Зона 2 | Зона 3 | Зона 4 | Зона 5 | Зона 6 |
|---|---|---|---|---|---|---|
| Легковые (≤8 мест) | 5 | 20 | 30 | 40 | 60* | 110* |
| Автобусы/микроавтобусы (>8 мест) | 5 | 15 | 25 | 30 | 60* | 90* |
| Спецтехника/грузовые | 5 | 10 | 20 | 20 | 60* | 70* |

`*` — у Зон 5/6: «по знакам/ПДД, но не выше». Доп. категории ТС (раздел 6): специальные, компрессорные установки.

**Правила приоритета (жёсткие, для движка):**
- **п. 7.3 — применяется НАИМЕНЬШЕЕ из (ПДД РК, внутренний регламент).**
- **п. 7.7 — нельзя превышать внутренний лимит, даже если ПДД допускают больше.**
- п. 7.4 — доп. снижение при дожде/тумане/гололёде/плохой видимости/перевозке ОГ (мягкий фактор).

### Механизм Б — Именованные геозоны с зашитым лимитом (стр. 16–27)
То, что **реально загружено в Omnicomm**. У каждой геозоны один лимит, привязанный к месту (в большинстве — независимо от типа ТС). **Это основной источник истины**; матрица (А) — фолбэк, когда точка вне именованной геозоны.

**Пороги нарушения (раздел 8):** до 3 км/ч — незначительное · 3–5 км/ч — существенное · **≥6 км/ч — грубое**. (Это ВНУТРЕННИЙ стандарт холдинга, строже КоАП РК, где штраф от 10 км/ч — см. [03](03-legal-framework-kz.md). Дисциплинарка по СТ КАП ≠ штраф по КоАП.)

## 2. Зависимость лимита от типа ТС / груза
Тип вшит в имя лишь у нескольких геозон KAP Logistics:
| Геозона | Лимит | Тип |
|---|---|---|
| Трасса Енбекши - Тайконыр (Легковые) | 80 | легковые |
| Трасса Енбекши - Тайконыр (Опасные грузы) | 50 | ОГ (−30 км/ч) |
| Трасса Каратау для легкового | 80 | легковые |
| Для легковых авто после спуска на перевал | 80 | легковые |

→ Один участок может иметь 2 геозоны с разным лимитом по категории. Модель: точка → кандидаты-геозоны → выбор по типу ТС, иначе min.

## 4. Классификация зон по имени (для авто-маппинга в Зоны 1–6)
- `ГТП`/`УГТП` → геотехнологические полигоны (Зона 4), 40–50
- `Тех(нолог). дорога` → технологические дороги (Зона 4), 20–50
- `Полигон`/`залежь`/`ЮК-…` → добычные (5–50; дренаж 5, шламонакопитель 10)
- `Завод`/`ОПЗ`/`АБК`/`Промплощадка` → промплощадки (Зона 1–2), 10–30
- `Вахтовый …` → вахтовые посёлки (Зона 1), 10–31
- `н.п.`/`г.`/`пос.`/`село` → населённые пункты (Зона 5), почти всегда 60
- `Трасса`/`Дорога до …`/`M-`/`A-`/`P-`/`R-` → трассы (Зона 6), 80–140
- `Знак ограничения …` / `СТОП` → точечные геозоны

**Диапазон:** мин 5 км/ч (дренаж РУ-6), макс 140 (магистрали). «Пересечённой местности» как термина в СТ нет — ближайшее = Зона 4 «Полигон, технологические дороги».

## 6. Выводы для движка `geozone_limit`
1. Источник истины — **именованные геозоны** (≈400+), матрица Таблицы 1 — фолбэк с разбивкой по 3 типам ТС.
2. Хранить лимит как `(геозона × тип_ТС)`, по умолчанию один лимит; для 4 KAP-зон — вариант по типу.
3. Координат нет → геометрию тянуть из Omnicomm по имени геозоны.
4. Конфликт: **min(внутренний, ПДД)**; внутренний нельзя превышать (п. 7.3/7.7).
5. Пороги нарушения СТ КАП: 3 / 3–5 / ≥6 км/ч (грубое) — отдельно от КоАП-штрафов.
6. Нужны **нормализация + дедупликация** (раздел 5) перед загрузкой как seed-справочник.

> **Следующий шаг (реализация):** собрать чистый seed-датасет `geozone → ДЗО → limit → [тип_ТС]` (CSV/JSON) из этого списка с исправлением аномалий, и матчить к геозонам Omnicomm по имени.
